# 기강 칭호 카탈로그

## 설계 원칙

- **조건 비공개**: 칭호 획득 조건은 UI에 노출하지 않음. 달성해서 알게 되는 방식.
- **획득 후 조건 표시**: 획득한 칭호는 카드 탭 시 하단 시트에서 조건 설명 확인 가능.
- **카테고리 내 최상위 1개만**: 러닝/철인/트레일/사이클은 최상위 등급 1개만 선택 가능.
- **등급(rarity_level) 비공개**: 등급은 내부 관리용 숫자(1~10). 유저에게 노출 안 함.
- **Event 칭호**: 단장 수여·이벤트용. 등급 개념 없음. 복수 보유·선택 가능. 일반 카운트에서 제외.
- **공식 대회 기록만 인정**: 러닝은 등록된 대회 기록 기준.

## 컬렉션 UI 구조

```
프로필 카드
└── [내 컬렉션] 버튼
    └── 바텀 시트
        ├── 탭: [ 칭호 ] [ 프레임 ]
        │
        ├── 칭호 탭
        │   ├── 일반  획득 12 / 24  ████████░░░░  50%
        │   ├── 칭호 그리드 (획득: 이펙트 풀, 미획득: ???  흐릿)
        │   │   └── 탭 시 하단 시트: 칭호명 + 이펙트 미리보기 + 획득 조건(획득 후만)
        │   │
        │   ├── Event  획득 2 / 5       ← 별도 섹션, 카운트 분리
        │   └── Event 그리드 (보유: 선택 가능, 미보유: 흐릿 — ??? 미표시)
        │
        └── 프레임 탭
            ├── 획득 3 / 12  ███░░░░░░░░░  25%
            └── 프레임 그리드 (획득: 선택 가능, 미획득: 흐릿)
```

### 칭호 선택 규칙
- **자동 계열(러닝 등)**: 상위 칭호 보유 시 하위는 선택 불가 (흐릿 + "상위 칭호 보유")
- **Event**: 보유한 것만 선택 가능, 복수 선택 가능
- **일반 카운트**: Event 제외, 자동 계열은 현재 최상위 1개만 카운트
