# 칭호 & 카드 프레임 시스템 설계

## 개요

멤버가 **칭호**와 **카드 프레임** 두 가지를 독립적으로 커스터마이즈할 수 있는 시스템.
- **칭호**: 관리자가 만들고 조건 달성 시 자동 부여 or 수동 수여. 이름 옆에 배지로 표시.
- **카드 프레임**: 멤버 본인이 직접 설정. 자기 이름이 나오는 모든 카드/행에 적용.

---

## 컬렉션 UI (내 컬렉션)

프로필 카드 안 **[내 컬렉션]** 버튼 → 바텀 시트로 열림.

### 탭 구조
```
[ 칭호 ] [ 프레임 ]
```

### 칭호 탭
- **일반** 섹션: 조건 달성으로 얻는 칭호
  - 획득 개수 / 전체 개수 + 진행률 바
  - 그리드 표시: 획득 → 이펙트 풀 적용, 미획득 → `???` + 흐릿
  - 자동 계열(러닝 등) 하위 칭호: 상위 보유 시 흐릿 + "상위 칭호 보유" 표시, 선택 불가
  - 탭 시 → 하단 시트: 칭호명 + 이펙트 미리보기 + 획득 조건(획득 후만 표시)
- **Special** 섹션: 단장 수여 칭호
  - 별도 카운트 (일반 카운트에서 제외)
  - 미보유 시 `???` 미표시, 흐릿 + "수여 필요"
  - 복수 선택 가능

### 프레임 탭
- 획득 개수 / 전체 개수 + 진행률 바
- 그리드: 획득 → 선택 가능, 미획득 → 흐릿
- 탭 시 → 하단 시트: 프레임 미리보기 + 선택 버튼

## 1. 칭호 시스템 (Title)

### 1-1. 데이터 모델 (현재 완성)

```
ttl_mst (칭호 마스터)
├── ttl_id
├── ttl_nm          — 칭호명 (예: "서브3", "철인", "100회 완주")
├── ttl_kind_enm    — auto(자동) | awarded(수동 수여)
├── ttl_ctgr_cd     — 카테고리 (running, triathlon, ...)
├── ttl_rank        — 등급 (숫자, 낮을수록 희귀)
├── effect_cd       — 배지 이펙트 ✅ 완성
├── cond_rule_json  — 자동 부여 조건 JSON
└── ...

mem_ttl_rel (멤버-칭호 관계)
├── is_prmy_yn      — 대표 칭호 여부 (딱 1개만 true)
└── ...
```

### 1-2. 칭호 배지 이펙트 (현재 완성)

| 코드 | 이름 | 느낌 |
|------|------|------|
| `none` | 없음 | 기본 스타일 |
| `neon` | 네온 글로우 | 사이버펑크 발광 |
| `hologram` | 홀로그램 | 보라/하늘/초록 프리즘 |
| `gold` | 골드 메탈릭 | 금속 광택 흐름 |
| `spark` | 스파클 | 별✦ 파티클 + 떠다님 |

### 1-3. 칭호 자동 부여 조건 (현재 완성)

| 조건 타입 | 설명 |
|-----------|------|
| `race_pb_under_sec` | 종목 PB가 N초 이하 |
| `race_finish_count` | 종목 완주 N회 이상 |
| `mileage_run_complete` | 마일리지런 목표 달성 |
| `attendance_count` | 출석 N회 이상 |
| `membership_days` | 가입 후 N일 이상 |

### 1-4. 표시 위치 (현재 완성)

- 프로필 카드 — 이름 옆 `sm` 사이즈
- 랭킹 페이지 — 이름 옆 `xs` 사이즈 (마라톤/트레일/철인3종)

### 1-5. 남은 작업

- [ ] 홈 최근 기록 카드에 칭호 배지 표시
- [ ] 프로필 탭에서 보유 칭호 목록 표시
- [ ] 대표 칭호 선택 UI (멤버가 직접)
- [ ] 프로젝트 참여자 목록에 칭호 표시

## 2. 카드 프레임 시스템 (Card Frame)

### 2-1. 개념

멤버 본인이 직접 설정하는 카드 외관 효과.
자기 이름이 포함된 **모든 카드/행**에 자동 적용.

### 2-2. 데이터 모델 (미구현)

```sql
-- team_mem_rel 또는 mem_mst에 컬럼 추가
ALTER TABLE team_mem_rel
  ADD COLUMN card_frame_cd text NOT NULL DEFAULT 'none';
  -- CHECK: none | neon | rainbow | gold | aurora | carbon | ...
```

AppMemberProfile 타입에도 `card_frame_cd` 필드 추가 필요.

### 2-3. 프레임 이펙트 후보

| 코드 | 이름 | 느낌 |
|------|------|------|
| `none` | 없음 | 기본 |
| `neon` | 네온 프레임 | 테두리 네온 발광 (칭호 neon과 세트) |
| `rainbow` | 레인보우 | 무지개 그라디언트 테두리가 회전 |
| `gold` | 골드 프레임 | 금테 테두리 반짝 |
| `aurora` | 오로라 | 배경에 파스텔 빛이 은은하게 흐름 |
| `carbon` | 카본 | 어두운 탄소섬유 질감 테두리 |
| `sakura` | 벚꽃 | 분홍 꽃잎이 배경에 흩날림 |

### 2-4. 적용 위치

| 위치 | 파일 | 컴포넌트 |
|------|------|---------|
| 홈 최근 기록 | `app/(main)/page.tsx` | `CardItem` per record |
| 랭킹 마라톤 | `records-client.tsx` | 각 행 div |
| 랭킹 트레일 | `records-client.tsx` | 각 행 div |
| 랭킹 철인3종 | `records-client.tsx` | 각 행 div |
| 프로필 카드 | `app/(main)/profile/page.tsx` | 최상단 `CardItem` |
| 프로젝트 참여자 | (미구현) | 참여자 목록 행 |

### 2-5. 구현 방식

`CardFrame` 래퍼 컴포넌트를 만들어 기존 카드/행을 감싸는 방식:

```tsx
// components/common/card-frame.tsx
<CardFrame frame={member.card_frame_cd}>
  <CardItem>...</CardItem>
</CardFrame>
```

CSS는 `globals.css`에 `.card-frame-*` 클래스로 정의.

### 2-6. 프레임 설정 UI (미구현)

- 위치: `/profile/edit` 또는 `/settings`
- 현재 프레임 미리보기 + 선택 그리드
- 선택 즉시 저장 (서버 액션)
