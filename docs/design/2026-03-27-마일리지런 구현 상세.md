## 1. 프로젝트 탭 레이아웃

### 1.1 하단 탭바 순서

```
홈 → 대회 → 프로젝트 → 랭킹 → 프로필
```

프로젝트 탭이 가운데(3번째)에 위치.

### 1.2 프로젝트 페이지 구조 (위→아래)

```
┌─────────────────────────────────┐
│ [프로젝트명]          [< 5월 >] │  ← 제목 + 월 네비게이터 (같은 줄)
├─────────────────────────────────┤
│ [참가 신청]                     │  ← 미참여자만 표시
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ 크루 진행현황 그래프        │ │  ← 마일리지/달성률 토글
│ │ N월 진행현황 통계           │ │  ← 같은 border 블록 안
│ └─────────────────────────────┘ │
├─────────────────────────────────┤
│ 내 현황                         │  ← 참여자만
├─────────────────────────────────┤
│ 환급 / 회식비                   │  ← 참여자만, 연습 기간이면 안내만
├─────────────────────────────────┤
│ 내 기록 (최신 5개 + 더보기)     │  ← 참여자만, 선택된 월 기록만
├─────────────────────────────────┤
│                          [+ FAB]│  ← 기록 입력 버튼
└─────────────────────────────────┘
```

### 1.3 접근 권한별 표시

| 상태 | 그래프+진행현황 | 참가신청 | 내현황/환급/기록 |
|------|:---:|:---:|:---:|
| 비로그인 | O | X | X |
| 로그인, 미참여 | O | O | X |
| 로그인, 신청 후 미승인 | O | O (대기 안내) | X |
| 로그인, 참여자 (승인됨) | O | X | O |

### 1.4 월 네비게이션

- **위치**: 제목 우측
- **형태**: `< 5월 >` 버튼
- **범위**: 시작월 1달 전(연습 기간) ~ 종료월
  - 예: 프로젝트 5월~9월 → 4월(연습)~9월까지 탐색 가능
- **구현**: `searchParams.month`로 서버 컴포넌트 재렌더

## 2. 연습 기간

- **정의**: 프로젝트 시작월 1달 전 (예: 5월 시작 → 4월이 연습)
- **연습 기간 특징**:
  - 기록 입력/조회 가능
  - 목표는 `initial_goal`과 동일
  - 환급/회식비 계산 **없음** → "N월은 연습 기간입니다" 안내 표시
  - 진행현황에 "(연습)" 라벨 표시
  - 환급 예상 총액, 회식비 풀 숨김

## 3. 참가비 구조 (기획서 변경)

기획서 원본과 다르게 확정된 구조:

| 항목 | 금액 | 비고 |
|------|------|------|
| 월별 보증금 | 잔여 개월 수 x 1만원 | 환급 대상 |
| 참가비 | **2만원** (기본) | 환급 없음 |
| 참가비 (싱글렛 보유) | **1만원** (1만원 할인) | 환급 없음 |

- 싱글렛비가 별도가 아니라, 참가비에서 싱글렛 보유 시 할인
- **회식비 풀 계산 시 참가비는 1인당 1만원 고정** (2만원 낸 사람의 나머지 1만원은 별도 적립)

## 4. 그래프 상세

### 4.1 X축 / Y축

- **X축**: 선택된 월의 1일 ~ 말일
- **Y축**: 마일리지(절대값) 또는 달성률(%)
- **토글 버튼**: "마일리지" / "달성률" (우측 상단)

### 4.2 데이터 표시

- **표시 대상**: 해당 월에 목표가 있거나 기록이 있는 참여자만
  - 7월 시작 참여자가 4월 그래프에 나오지 않음
- **누적 마일리지**: 각 일자까지의 누적합
- **달성률**: `min(누적 마일리지 / 목표, 100)%` — 100%가 최대
- **목표 기준선**:
  - 마일리지 모드: 본인 목표값 수평 점선
  - 달성률 모드: 100% 수평 점선

### 4.3 툴팁

- 상위 5명만 표시 (값 내림차순)
- 본인이 상위 5명에 없으면 추가로 표시
- 나머지는 "외 N명"으로 축약

### 4.4 그래프 갱신

- 기록 입력/수정/삭제 후 `router.refresh()` + `refreshKey={Date.now()}`로 즉시 반영

## 5. N월 진행현황 통계

그래프와 같은 border 블록 안에 위치. 그래프 모드(마일리지/달성률)에 연동.

| 항목 | 마일리지 모드 | 달성률 모드 |
|------|:---:|:---:|
| 참여 인원 | N명 | N명 |
| 평균 값 | **평균 마일리지** (km) | **평균 달성률** (%) |
| 달성 인원 | N / N명 | N / N명 |
| 환급 예상 총액 | O (실전만) | O (실전만) |
| 회식비 풀 | O (실전만) | O (실전만) |

- **참여 인원**: 해당 월 이전에 참가 신청한 전원 (목표 유무 무관, `start_month <= 해당 월`)
- **평균 달성률**: 각 참여자의 `min(달성/목표, 1)` 평균 × 100
- **목표 fallback**: `mileage_goal`이 없으면 `initial_goal` 사용
- **ChartModeContext**: 그래프와 진행현황이 mode를 공유하는 React Context

## 6. 환급 / 회식비

### 6.1 표시 항목

| 좌측 | 우측 |
|------|------|
| **확정 환급액** (현재 월 포함) | **회식비 지원 예상금액** (1인 상한) |

하단:
- **회식비 잔여 총액** (전체 풀)
- "내 지분: 참여 N개월 / 전체 N개월"

### 6.2 확정 환급액

```
확정 환급액 = Σ (실전 기간 각 월의 환급액)
월 환급률 = min(달성 마일리지 / 목표, 1.0)
월 환급액 = 월 환급률 × 1만원
```

- **실전 기간만** 계산 (연습 기간 제외)
- 당월 예상분도 포함

### 6.3 회식비 풀

```
회식비 풀 = (전원 보증금 합 - 전원 환급액 합) + 전원 참가비(1인 1만원)
```

- 연습 기간 목표는 제외
- 싱글렛 보유 여부와 무관하게 참가비는 **1인당 1만원** 고정

### 6.4 1인 회식비 지원 예상

```
1인 지분 = 본인 참여 개월 수 (실전 기간)
1인 상한 = 회식비 풀 × (본인 지분 / 전체 참여자 지분 합)
```

### 6.5 연습 기간

"N월은 연습 기간입니다. 환급 및 회식비는 실전 기간(5월~)부터 적용됩니다."

## 8. 내 기록

- **즉시 반영**: 입력/수정/삭제 후 `router.refresh()`

## 9. 월별 목표 자동 생성

### 9.1 방식

**lazy + 일괄 생성**: 아무 참여자가 프로젝트 페이지에 접속하면 전체 참여자의 당월 목표를 일괄 생성.

### 9.2 로직 (`ensureAllCurrentMonthGoals`)

1. 프로젝트의 전체 참여자 중 `start_month <= 당월`인 사람 조회
2. 각 참여자에 대해:
   - 당월 `mileage_goal`이 이미 있으면 스킵
   - 이전 월 목표가 있으면 → 달성 여부 확인 → `calcNextMonthGoal` 규칙 적용
   - 이전 월 목표가 없으면 → `initial_goal`로 생성

### 9.3 목표 갱신 규칙

| 당월 목표 범위 | 달성 시 다음달 | 미달성 시 |
|:---:|:---:|:---:|
| 0~49 | +10 | 유지 |
| 50~99 | +15 | 유지 |
| 100 이상 | +20 | 유지 |

## 10. 참가 신청

### 10.1 목표 선택

| 옵션 | 값 |
|------|-----|
| 초보 | 50 km/월 |
| 고수 | 100 km/월 |
| 초고수 | 자유 입력 (1 이상) |

### 10.2 보증금 계산

- **잔여 개월 수**: `max(프로젝트 시작월, 현재 월)` ~ 종료월 (동적 계산)
- **보증금**: 잔여 개월 × 1만원
- **참가비**: 싱글렛 보유 시 1만원, 미보유 시 2만원
- **합계**: 보증금 + 참가비

### 10.3 승인 흐름

1. 앱에서 참여하기 → `project_participation` 생성 (`deposit_confirmed = false`)
2. 관리자가 `/admin/participations`에서 입금 확인 → `deposit_confirmed = true`

## 11. 관리자 기능

### 11.1 대시보드 (`/admin`)

두 섹션으로 분리:

**일반**:
- 가입 승인 대기
- 활성 회원
- 이번 달 대회
- 전체 기록

**프로젝트**:
- 참여자 관리 (대기 N / 참여 N)
- 활성 프로젝트
- 활성 이벤트

### 11.2 참여자 관리 (`/admin/participations`)

**승인 대기 섹션**:
- 미승인 참여자 목록
- 각 항목: 이름, 초기 목표, 납부 금액 (보증금 + 참가비 상세), 싱글렛 할인 표시
- 승인/거절 버튼
- 승인 시 즉시 참여자 목록으로 이동

**참여자 목록 섹션**:
- 승인된 참여자 전체 목록
- 이름, 초기 목표, 시작월, 싱글렛 보유 여부, 납부 금액

### 11.3 이벤트 배율 관리 (`/admin/events`)

- CRUD: 이름, 배율, 시작일/종료일, 활성 토글
- 기존 기록 소급 변경 없음 (입력 시점 배율 적용)

### 11.4 프로젝트 관리 (`/admin/projects`)

- CRUD: 이름, 시작월, 종료월, 상태(active/ended) 토글

### 11.5 설정 페이지

- 관리자에게만 "관리자 페이지" 단일 링크 표시 (`/admin`)
