# 5탭 UX 평가 리포트 (2026-07-03)

> 하단 탭 5개(홈/기강이야기/프로젝트/랭킹/프로필)에 대한 UX 디자인 평가.
> 디자이너·접근성 관점 직접 리뷰 + "기강 멤버" 서비스 페르소나 교차 리뷰(saas-dev-loop T4).
> 토스(앱인토스 UX 가이드) 등 국내 우수사례 기준과 비교. **이 문서를 기반으로 항목별로 조금씩 최적화한다.**
> 완료한 항목은 체크하고, 결정이 바뀐 항목은 취소선 + 사유를 남길 것.

## 총평

로딩·인터랙션 디테일(스켈레톤, 낙관적 UI, 인스턴트 오픈, 딥링크, motion-reduce 대응)은 이미 토스급.
반면 **정보 구조(탭 구성·레이블링)와 "나 중심 개인화"**에서 점수를 잃는다.
"탭을 눌렀을 때 기대한 게 나오는가"라는 1차 관문에서 걸리는 지점이 여럿 있다.

| 영역 | 평가 | 근거 |
|------|------|------|
| 로딩 UX | ✅ 토스급 | 전 탭 Suspense + 형태 맞춘 스켈레톤, 등록 직후 쿼리 생략 인스턴트 오픈 |
| 모션 절제 | ✅ 좋음 | 그라데이션·글로우 없음, `active:scale-[0.98]`, motion-reduce 대응 |
| 디자인 시스템 규율 | ⚠️ 중간 | 토큰·타이포 체계는 있으나 캘린더/랭킹에서 이탈 다수 |
| UX 라이팅 | ❌ 미흡 | 토스는 전 문구 해요체 강제 — 해요체/격식체 혼용 잔존 |
| 개인화 | ❌ 미흡 | 랭킹 내 위치, 탭 배지, 미등록자 일정 티커 등 "나 중심" 장치 부족 |
| 정보 구조 | ❌ 미흡 | 죽은 탭, 라벨-콘텐츠 불일치, 고아 라우트 |

## P0 하이라이트

### P0-1. 죽은 탭이 내비게이션의 20%
- 위치: `components/bottom-tab-bar.tsx:11`, `app/(main)/story/page.tsx`
- 문제: "기강이야기" 탭은 언제 눌러도 "곧 오픈됩니다" placeholder. 다른 탭과 동일한 시각적 무게로 없는 기능을 약속한다. 토스 원칙: 진입 순간 목적을 수행할 수 있어야 하고, 모든 탭은 즉시 가치를 줘야 한다.
- 개선: 오픈 전까지 탭 숨김(4탭 구성), 또는 최소한 "준비중" 표기로 기대치 조정.
- [ ] 처리

### P0-2. 랭킹에서 "나"를 찾을 수 없음
- 위치: `app/(main)/records/page.tsx:36` → `records-client.tsx`
- 문제: `RecordsClient`에 현재 멤버 id가 전달되지 않아 내 행 하이라이트/점프 불가. 자기 순위를 보려면 검색창에 자기 이름을 타이핑해야 한다. 개인화 효용이 가장 큰 화면인데 "나"가 없다.
- 개선: 서버에서 memId 전달 → 내 행 "나" 배지 + 배경 강조 + "내 순위로 이동" 버튼.
- [ ] 처리

### P0-3. 판독 불가 텍스트 + 미달 터치 타깃
- 위치: `components/home/mini-calendar.tsx:1306`(이벤트 바 `text-[7px]`), `:1147-1162`(월 네비 `size-5`=20px), `:1169-1190`(필터 칩 ~26px), `app/(main)/profile/page.tsx:196-198`(설정 링크 22px)
- 문제: WCAG 2.2 최소 24px·모바일 권장 44px 미달. 러닝 직후 손으로 조작하는 앱이라 타깃 크기가 더 중요하다. 7px 텍스트는 사실상 색 막대 역할만 한다.
- 개선: 이벤트 바 9px+ 또는 색상 점 전용으로 단순화. 버튼류는 히트 영역만이라도 44px 근접(패딩 확장).
- [ ] 처리

## 탭별 상세

### 홈 (`/`)
캘린더 중심 구조 + 헤더 티커는 좋은 선택. D-5 모임을 슬로건과 교차 롤링하는 티커(`components/home/header-ticker.tsx`)는 토스식 "한 줄 개인화" 그대로.

- [ ] **[P1] 티커가 RSVP한 모임만 감지** — 미등록 멤버의 "이번 주말 뭐 있지?"에 무용. 팀 일정 전체를 대상으로 하고, 내가 등록한 건 강조를 달리하는 방식 검토 (`app/(main)/page.tsx:28-40`)
- [ ] **[P2] 캘린더 색상 범례 없음** — 대회(주황)/모임(보라)/정보(파랑)의 의미를 알 방법이 없음. 필터 칩에 색상 점 부착으로 칩↔캘린더 색 연결 (`mini-calendar.tsx:1169-1190`)
- [ ] **[P2] 비로그인 홈에 가입 CTA 부재** — 유일한 동선이 최하단 SocialLinksGrid 카카오 아이콘. 비로그인 상태 상단에 가입 배너/카드 검토
- [ ] **[P2] 매직넘버 타이포 다수** — `text-[6~13px]`, `text-[18px]` 산재. 타이포 컴포넌트/토큰 체계로 수렴 (DESIGN.md 규칙 #1)
- [ ] **[P2] 모임 색상 `violet-500` 하드코딩** — 일정유형 토큰 신설 후 승격 (DESIGN.md 규칙 #10)

### 기강이야기 (`/story`)
- [ ] **[P0-1]** 위 참조 — 탭 숨김 또는 준비중 표기
- [ ] **[P1] `/races` 고아 라우트 처리** — 대회 탭이 빠진 뒤 앱 내 진입점 0개(스토리북만 참조). 재진입 동선을 만들거나 페이지 제거 결정 (`app/(main)/races/page.tsx`)
- [ ] **[P2] 해요체/격식체 혼용** — "곧 오픈됩니다."(격식) + "만나볼 수 있어요"(해요체)가 한 블록. 전역 해요체 통일 기준으로 빈 상태 문구 일괄 점검

### 프로젝트 (`/projects`)
- [ ] **[P1] 비로그인 redirect에 `?next=/projects` 누락** — 로그인 후 홈으로 떨어짐. 프로필(`?next=/profile`)과 동일 패턴으로 통일 (`app/(main)/projects/page.tsx:31`)
- [ ] **[P1] 탭 라벨 "프로젝트"로 마일리지런 예측 불가** — 라벨 재검토("마일리지" 등), 또는 멀티 프로젝트 로드맵 확정 시 유지
- [ ] **[P2] 크루/나 스코프 교차 배치** — 크루 차트→내 현황→랜덤 후기→크루 통계→내 환급/차트/목록 순 단일 스크롤 8섹션. "내 현황" 그룹 우선 배치 또는 세그먼트 분리 검토 (토스 One Thing per One Page)
- [ ] **[P2] 비로그인 시 무설명 로그인 리다이렉트** — 탭 안에서 "로그인하면 마일리지를 기록할 수 있어요" 안내 + 로그인 버튼이 더 부드러움 (프로필 탭 공통)

### 랭킹 (`/records`)
- [ ] **[P0-2]** 위 참조 — 내 순위 표시
- [ ] **[P2] 탭 라벨 "랭킹" vs 페이지 제목 "기강의 전당" 불일치** — 통일 또는 병기
- [ ] **[P2] 메달 색 하드코딩** — `text-amber-500`/`slate-400`/`amber-700` → 토큰화 (규칙 #10)
- [ ] **[P2] 빈 상태에 기록 등록 동선 없음** — "프로필에서 내 기록 등록하기" 링크 추가 + 자체 EmptyState 재정의를 공통 컴포넌트로 교체 (규칙 #4)
- [ ] **[P2] 철인3종 "킹" 라벨** — 크루 은어. "킹(풀코스)" 병기 검토 (내부 문화상 유지도 가능 — 결정 필요)

### 프로필 (`/profile`)
구조(프로필 카드 → 바로가기 4종 → PB 그리드 → 페이스 차트)는 깔끔. 회비 미납 빨간 점은 좋은 디테일.

- [ ] **[P1] 회비 미납 배지가 프로필 탭 안에서만 보임** — 탭바 아이콘 배지로 승격 검토 (토스: 배지는 필수 알림에만 — 미납은 해당)
- [ ] **[P2] 설정 아이콘 링크에 접근 가능한 이름 없음** — `aria-label="설정"` 추가 (`app/(main)/profile/page.tsx:196-198`)
- [ ] **[P2] 바로가기 4종의 낮은 어포던스** — 14px 아이콘 + text-xs 전부 muted — 시각 위계 재검토

### 탭바 공통 (`components/bottom-tab-bar.tsx`)
- [ ] **[P1] `aria-current="page"` 부재** — active 탭이 색으로만 구분
- [ ] **[P1] 히트 영역이 콘텐츠 크기뿐** — `px-3 py-1`이라 탭 셀 사이가 데드존. 각 Link를 `flex-1 h-full`로 확장
- [ ] **[P2] `prefetch={false}` 재검토** — 전 페이지 Suspense 스켈레톤 완비 상태라 prefetch 허용 시 탭 전환 체감 개선 여지
- [ ] **[P2] Zap(⚡) 아이콘 중복 의미** — 프로젝트 탭 vs 헤더 "업데이트" 게시판. 한쪽 교체
- [ ] **[P1] 페이지 헤더 불일치** — 대회/랭킹/프로필이 raw `h-14` div + `H1 font-semibold` 오버라이드, 스토리/프로젝트만 PageHeader, 홈은 h-20. PageHeader로 통일 + 높이 관계 정리 (규칙 #2)

## 권장 착수 순서

1. **기강이야기 탭 숨김(4탭) or 준비중 표기** — 체감 완성도 즉시 상승, 변경 범위 최소
2. **랭킹 내 순위 표시** — 개인화 효용 최대 지점
3. **터치 타깃·7px 텍스트 일괄 보정** — 접근성 + 일상 사용성
4. 탭바 접근성/히트영역/헤더 통일 — 한 PR로 묶기 좋음
5. 라이팅(해요체)·라벨 정합 일괄 스윕
6. 나머지 P2 개별 처리

## 리뷰 방법 메모

- 코드 기반 정적 리뷰 (5탭 페이지 + `bottom-tab-bar` + `mini-calendar` 렌더부 + `records-client` + `header-ticker` 등)
- 서비스 페르소나 "기강 멤버"(sonnet 서브에이전트) 시나리오 리뷰: ①주말 일정 확인 ②내 기록 조회 ③내 랭킹 위치 ④이달 마일리지 ⑤탭 레이블 예측성 ⑥카피 톤 ⑦비로그인 첫인상
- 스크린샷/실기기 검증은 미수행 — 시각 대비(색 대비 4.5:1 등)는 구현 시 재확인 필요
