# 칭호 시스템 설계

## 개요

기강 회원의 스포츠 활동 이력에 기반한 칭호(뱃지) 시스템. 러닝, 철인, 트레일러닝, 자전거 4개 종목 자동 칭호와 기강단장이 수여하는 기간제 칭호로 구성된다.

## 공통 규칙

- **모든 자동 칭호**는 기강 가입일(`member.joined_at`) 이후의 공식 대회 기록만 인정
- `race_result`에 기록이 존재하면 완주로 간주 (별도 DNF 판별 없음)
- 이름 이펙트의 구체적 디자인은 추후 별도 정의

## 칭호 카테고리

### 1. 러닝 (`running`)

카테고리 내 최상위 등급 1개만 보유. 가입 이후 공식 대회 기록만 인정.

| 등급 | 칭호명 | 조건 |
|------|--------|------|
| 1 | 런린이 | 가입 3개월 이하이면서 상위 칭호 조건 미충족 |
| 2 | 입문 | 가입 3개월 초과이면서 상위 칭호 조건 미충족 |
| 3 | 초보 | 10K PB 1시간 이내 (record_time_sec <= 3600) |
| 4 | 중수 | 하프 PB 2시간 이내 (record_time_sec <= 7200) |
| 5 | 고수 | 풀 PB 5시간 이내 (record_time_sec <= 18000) |
| 6 | 고인물 | 풀 PB 4시간 이내 (record_time_sec <= 14400) |
| 7 | 신세계 | 풀 PB 3시간 30분 이내 (record_time_sec <= 12600) |
| 8 | 천상천하유아독존 | 풀 PB 3시간 10분 이내 (record_time_sec <= 11400) |
| 9 | 최고존엄 | 풀 PB 3시간 이내 (record_time_sec <= 10800) |

- 상위 등급 달성 시 하위 자동 대체 (하위 건너뛰기 가능)
- 런린이/입문은 "기본 칭호"로, 상위 조건을 충족하지 못하면 가입 기간에 따라 자동 부여됨 (대회기록이 있어도 조건 미달이면 런린이/입문 유지)
- 모든 시간 조건은 "이내" = 해당 초 포함 (`<=`)
- 이모지: 달리기 아이콘

### 2. 철인 (`triathlon`)

완주 기록 존재 여부로 판정 (`race_result`에 기록이 있으면 완주로 간주). 카테고리 내 최상위 1개만 보유.

| 등급 | 칭호명 | 조건 |
|------|--------|------|
| 1 | 올림픽 철인 | TRIATHLON_OLYMPIC 완주 기록 있음 |
| 2 | 하프철인 | TRIATHLON_HALF 완주 기록 있음 |
| 3 | 킹철인 | TRIATHLON_FULL 완주 기록 있음 |

- 이모지: 메달 아이콘 (금/은/동 등)

### 3. 트레일러닝 (`trail`)

트레일 대회 완주 기록 존재 여부로 판정. 카테고리 내 최상위 1개만 보유.

> **백엔드 참고**: 현재 `race_result`에 `sport` 컬럼이 없어서 트레일 기록과 울트라마라톤 기록(50K/100K/100M 등)을 구분할 수 없음. `race_result`에 `sport` 컬럼 추가 또는 `competition_id` FK 추가 등의 스키마 변경이 필요.

| 등급 | 칭호명 | 조건 |
|------|--------|------|
| 1 | 동네언덕 | 트레일 대회 참가 기록 있음 |
| 2 | 뒷산 | 20K 트레일 완주 기록 있음 |
| 3 | 산악구보 | 50K 트레일 완주 기록 있음 |
| 4 | 산악대장 | 100K 트레일 완주 기록 있음 |
| 5 | 산신령 | 100M 트레일 완주 기록 있음 |

- 이모지: 산 아이콘

### 4. 자전거 (`cycling`)

완주 기록 존재 여부로 판정. 카테고리 내 최상위 1개만 보유.

> **백엔드 참고**: 현재 cycling event_type에 `MEDIOFONDO`가 없음 (`GRANFONDO`, `ROAD RACE`, `TIME TRIAL`, `CRITERIUM`만 존재). `MEDIOFONDO` event_type 추가가 필요.

| 등급 | 칭호명 | 조건 |
|------|--------|------|
| 1 | 메디오폰도 | MEDIOFONDO 완주 기록 있음 |
| 2 | 그란폰도 | GRANFONDO 완주 기록 있음 |

- 이모지: 🚲(메디오폰도) / 🚴(그란폰도) — 플랫티콘 등 별도 아이콘으로 대체 가능

### 5. 기강단장 수여 (`awarded`)

기강단장(모임장)이 직접 수여하는 기간제 칭호. **복수 보유 가능**.

| 칭호명 | 설명 |
|--------|------|
| 서브현근 | 모임장보다 풀코스 빠름 |
| 기강단장 | 모임장 |
| 행동대장 | 운영진 |
| 맛객 | 맛집을 잘 앎 |
| (추가 가능) | 단장이 자유롭게 생성 |

- 수여 시 기간 직접 지정 (예: 3개월, 6개월)
- 만료 시 자동으로 표시 안 됨 (데이터는 보존)
- 포인트는 수여 시 수동 입력

## 포인트 시스템

### 자동 칭호 포인트 (희소성 기반)
```
칭호 포인트 = 전체 활성 회원 수 / 해당 칭호 보유자 수
```

- 예: 회원 100명, 고인물 10명 → 고인물 인당 10점
- 예: 회원 100명, 런린이 100명 → 런린이 인당 1점
- 회원 수 변동 시 실시간(조회 시) 재계산

### 수여 칭호 포인트
- 수여 시 단장이 직접 점수 입력

### 회원 총 포인트
```
총 포인트 = 보유 중인 모든 칭호 포인트 합산
```

- 정렬이 필요한 곳에서 총 포인트 내림차순 사용

## 이름 표시 규칙

### 표시 순서
```
홍길동 🏃 ⛰️ 🏅 🚴 [맛객] [행동대장]
```

- 이름 뒤에 자동 칭호 이모지 → 수여 칭호 뱃지
- **포인트 높은 칭호가 앞**에 위치
- 보유 카테고리만 표시

### 적용 화면

| 화면 | 표시 방식 | 이름 이펙트 | 포인트 정렬 |
|------|-----------|------------|------------|
| 프로필 | 자세히 보기 (기본) | O | X |
| 기록 랭킹 | 축약 → 탭하면 자세히 | O | O (선택) |
| 대회 참가자 목록 | 이름만 | O (이펙트만) | X |
| 홈 | 축약 | O | X |

### 자세히 보기 (프로필 / 랭킹 탭 시)
- 보유 칭호 전체 목록 (포인트 높은 순)
- 카테고리별 칭호명 + 획득 조건/일시
- 수여 칭호는 만료일도 표시

### 이름 이펙트 (디자인팀 참고)
- **러닝 칭호 보유**: 이름 폰트 스타일 변화
- **철인 칭호 보유**: 이름 둘레 장식
- **트런 칭호 보유**: 테두리 후광 효과
- **자전거 칭호**: 디자인팀이 정의
- **수여 칭호**: 디자인팀이 정의

## 칭호 자동 계산 & 갱신

### 재계산 시점
- 기록 등록/수정/삭제 시: 해당 회원의 칭호 재계산
- 회원 가입 시: 런린이 자동 부여
- 가입 3개월 경과 시: 런린이 → 입문 전환 (배치 또는 조회 시 계산)

### 계산 로직
- **러닝**: `race_result`에서 가입일(`member.joined_at`) 이후 기록만 필터 → 조건 충족하는 최상위 등급 적용
- **철인/트런/자전거**: `race_result`에서 가입일 이후 해당 event_type 완주 기록 존재 여부 → 최상위 등급 적용
- 각 카테고리 내 기존 칭호를 새 등급으로 교체

### 수여 칭호 관리
- 기강단장이 관리 화면에서 수여 (대상 회원, 칭호명, 기간, 포인트 지정)
- 만료된 칭호는 자동으로 표시 안 됨 (데이터는 보존)

## DB 스키마 (참고용)

백엔드 팀에서 상세 설계. 아래는 방향성 참고.

### `title_definition` (칭호 정의)
- id, category, name, rank, condition_type(auto/manual), condition_rule(jsonb), emoji, base_point

### `member_title` (회원별 보유 칭호)
- id, member_id(FK), title_id(FK), granted_at, expires_at(nullable), granted_by(nullable FK)

### 포인트
- 자동 칭호: 조회 시 `활성 회원 수 / 보유자 수`로 동적 계산
- 수여 칭호: `title_definition.base_point` 사용
