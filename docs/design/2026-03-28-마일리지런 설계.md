# 마일리지런 프로젝트 설계서

> 기강 러닝크루 앱 — "프로젝트" 탭 첫 번째 프로젝트
> 최종 업데이트: 2026-03-28

---

## 1. 개요

기강 러닝크루 앱에 **프로젝트 탭**을 신규 추가하고, 첫 번째 프로젝트로 **마일리지런**을 운영한다.

- **프로젝트 탭**: 하단 탭바 가운데(3번째)에 위치. 항상 활성 프로젝트 1개가 바로 표시됨.
- **마일리지런**: 5월~9월(5개월) 동안 크루원 각자 목표 마일리지를 설정하고, 매월 달성하며 점진적으로 목표를 높여가는 이벤트.
- **연습 기간**: 4월(시작월 1달 전)은 연습 기간. 기록 입력/조회 가능하나 환급/회식비 계산 없음.
- 이 탭은 범용이며, 시즌에 따라 다른 프로젝트(예: 동계훈련)로 교체될 수 있음.

### 기술 스택

Next.js (App Router) + Supabase + Recharts + shadcn/ui + pnpm. 기존 패턴을 따를 것.

### 하단 탭바 순서

```
홈 → 대회 → 프로젝트 → 랭킹 → 프로필
```

### 2.2 접근 권한별 표시

| 상태 | 그래프+진행현황 | 참가신청 | 내현황/기록 | 규칙 버튼 |
|------|:---:|:---:|:---:|:---:|
| 비로그인 | O | X | X | O |
| 로그인, 미참여 | O | O | X | O |
| 로그인, 신청 후 미승인 | O | O (대기 안내) | X | O |
| 로그인, 참여자 (승인됨) | O | X | O | O |

### 2.3 월 네비게이션

- **위치**: 제목 우측
- **형태**: `< 5월 >` 버튼
- **범위**: 시작월 1달 전(연습 기간) ~ 종료월
  - 예: 프로젝트 5월~9월 → 4월(연습)~9월까지 탐색 가능
- **구현**: `searchParams.month`로 서버 컴포넌트 재렌더
- 모든 하위 컴포넌트(그래프, 진행현황, 내현황, 기록 등)에 선택된 월을 전달

## 3. 연습 기간

- **정의**: 프로젝트 시작월 1달 전 (예: 5월 시작 → 4월이 연습)
- **연습 기간 특징**:
  - 기록 입력/조회 가능
  - 목표는 `initial_goal`과 동일
  - 환급/회식비 계산 **없음** → "N월은 연습 기간입니다" 안내 표시
  - 진행현황에 "(연습)" 라벨 표시
  - 총 참가비, 총 회식비, 총 환급금 숨김

## 4. 마일리지 계산 규칙

### 4.1 종목별 기본 마일리지

| 종목 | 계산식 |
|------|--------|
| 러닝 | 거리(km) × 1 + 상승고도(m) / 100 |
| 트레일러닝 | 거리(km) × 1 + 상승고도(m) / 100 |
| 자전거 | 거리(km) / 4 + 상승고도(m) / 100 |
| 수영 | 거리(km) × 3 (상승고도 없음) |

- 거리 단위: km
- 상승고도 단위: m (수영 제외 전 종목 입력 가능)

### 4.2 이벤트 배율

운영진이 관리자 페이지(`/admin/events`)에서 추가/수정/삭제할 수 있는 배율 이벤트:

| 이벤트 예시 | 배율 |
|------------|------|
| 비 올 때 | ×1.2 |
| 모임에서 뛰기 | ×1.1 |

- 이벤트별 시작일/종료일 설정 가능
- 이벤트는 **중복 적용** 가능 (곱셈)
- 기록 입력 시 현재 활성 상태인 이벤트를 체크박스로 선택
- 이벤트가 비활성화되어도 기존 기록의 마일리지는 소급 변경하지 않음
- 이벤트 배율이 변경되어도 기존 기록에는 영향 없음 (입력 시점의 배율 스냅샷 저장)

### 4.3 최종 마일리지 계산

```
최종 마일리지 = 기본 마일리지 × 이벤트배율1 × 이벤트배율2 × ...
```

### 4.4 소수점 처리

- **저장**: 소수점 둘째자리까지 (`Math.round(x * 100) / 100`)
- **표시**: 소수점 첫째자리까지

## 5. 목표 및 월별 진행 규칙

### 5.1 초기 목표 설정

| 옵션 | 값 |
|------|-----|
| 초보 | 50 km/월 |
| 고수 | 100 km/월 |
| 초고수 | 자유 입력 (1 이상) |

### 5.2 월별 목표 자동 갱신

목표는 정수값으로 관리한다.

| 당월 목표 범위 | 달성 시 다음달 | 미달성 시 다음달 |
|:---:|:---:|:---:|
| 0~49 | +10 | 유지 |
| 50~99 | +15 | 유지 |
| 100 이상 | +20 | 유지 |

- **달성 판정**: 해당 월 1일~말일 기록의 마일리지 합산 >= 해당 월 목표
- **달성률 최대**: 100% (초과 달성해도 100%로 cap)

### 5.3 목표 자동 생성 방식

**DB 트리거 + lazy 일괄 생성** 이중 구조:

#### 5.3.1 DB 트리거 (실시간)

`activity_log` 테이블에 INSERT/UPDATE/DELETE 발생 시 PostgreSQL 트리거가 자동 실행:

1. 해당 기록이 속한 월의 목표를 조회
2. 해당 월의 전체 마일리지를 합산
3. 달성 여부 판정 → 다음 달 목표를 계산 (갱신 규칙 적용)
4. 다음 달 목표를 upsert (없으면 생성, 있으면 갱신)

이를 통해:
- 기록 입력 시 목표 달성되면 → 다음 달 목표 자동 상향
- 기록 삭제로 미달성 전환되면 → 다음 달 목표 자동 하향 (원래 값으로)

서버 액션 성능에 영향 없음 (DB 레벨에서 처리).

#### 5.3.2 lazy 일괄 생성 (안전망)

아무 참여자가 프로젝트 페이지에 접속하면 전체 참여자의 당월 목표를 일괄 생성:

1. 프로젝트의 전체 참여자 중 `start_month <= 당월`인 사람 조회
2. 각 참여자에 대해:
   - 당월 `mileage_goal`이 이미 있으면 스킵
   - 이전 월 목표가 있으면 → 달성 여부 확인 → 갱신 규칙 적용
   - 이전 월 목표가 없으면 → `initial_goal`로 생성

### 5.4 목표 수정

- 해당 월 **1일~14일(포함)** 까지만 수정 가능. 15일부터 잠김.
- 현재 목표 **이상으로만** 수정 가능 (하향 불가)

### 5.5 목표 fallback

`mileage_goal`이 없는 월에는 `initial_goal`(참가 신청 시 설정한 값)을 표시/계산에 사용.

## 6. 참여 신청 및 보증금

### 6.1 참여 신청

- 앱 내 "참여하기" 버튼으로 신청 (제목과 그래프 사이에 위치, 미참여자만 표시)
- 신청 시 **마일리지런 규칙이 상단에 인라인으로 표시**됨
- 신청 시 초기 목표 설정 (초보 50 / 고수 100 / 초고수 자유입력)
- 운영진이 보증금 입금 확인 후 참여 승인

### 6.2 보증금 구성

| 항목 | 금액 | 비고 |
|------|------|------|
| 월별 보증금 | 잔여 개월 수 × 1만원 | 환급 대상 |
| 참가비 | **2만원** (기본) | 환급 없음 |
| 참가비 (싱글렛 보유) | **1만원** (1만원 할인) | 환급 없음 |

- **잔여 개월 수**: `max(프로젝트 시작월, 현재 월)` ~ 종료월 (동적 계산, 최대 5개월)
- **회식비 풀 계산 시 참가비는 1인당 1만원 고정** (2만원 낸 사람의 나머지 1만원은 별도 적립)

**예시**: 5월 시작 + 싱글렛 미보유 = 5만(보증금) + 2만(참가비) = **7만원**
**예시**: 5월 시작 + 싱글렛 보유 = 5만(보증금) + 1만(참가비) = **6만원**

### 6.3 중도 참여

- 언제든 합류 가능
- 월 중 합류해도 해당 월부터 시작 (5월 10일 합류 → 5월부터)
- 중도 합류자도 합류 월의 전체 목표를 달성해야 함 (일할 계산 없음)
- 보증금은 잔여 개월분 기준 (합류 월 포함)

### 6.4 탈퇴/포기

- **중도 탈퇴 기능 없음** — 앱에서 탈퇴 버튼 없음
- 마일리지런은 중도 포기가 없음. 부상, 이사 등 어떤 사유에도 중도 포기되지 않음.
- 사실상 포기자 = 활동을 안 하는 사람. 달성률 0%인 월은 미환급 → 회식비 풀로 전환
- 환급은 9월 종료 시 일괄만, 예외 없음

### 6.5 승인 흐름

1. 앱에서 참여하기 → `project_participation` 생성 (`deposit_confirmed = false`)
2. 관리자가 `/admin/participations`에서 입금 확인 → `deposit_confirmed = true`

## 7. 환급 계산

### 7.1 월별 환급

```
월 환급률 = min(해당 월 달성 마일리지 / 해당 월 목표, 1.0)
월 환급액 = 월 환급률 × 1만원
```

- 초과 달성해도 환급률은 최대 1.0 (100%)
- **실전 기간만** 계산 (연습 기간 제외)

### 7.2 총 환급액

```
총 환급액 = 각 실전 참여 월의 환급액 합산 (누적)
```

**예시**: 5개월 참여, 달성률 100%, 100%, 100%, 70%, 30%
→ (1.0 + 1.0 + 1.0 + 0.7 + 0.3) × 1만 = **4만원 환급**

### 7.3 환급 시점

- 9월 종료 시 일괄 환급

### 7.4 앱 표시 (내 현황 카드 하단)

| 좌측 | 우측 |
|------|------|
| **내 환급금** (누적) | **내 회식지원비** (예상) |

하단 안내:
- 환급금은 이벤트 종료 후 일괄 환급됩니다.
- 회식지원금은 예상금액으로 참여인원수에 따라 달라집니다.

## 8. 회식비 계산

### 8.1 총 참가비

```
총 참가비 = Σ(각 월별 활성 참가 인원 × 보증금 1만원) + 활성 참가자 수 × 참가비 1만원
```

- **활성 참가 인원**: 해당 월 기준 `start_month <= 해당 월`이고 `deposit_confirmed = true`인 참가자
- **보증금 개월 수**: 프로젝트 시작월부터 카운트 (시작 전 가입자도 시작월부터)
- **참가비**: 활성 참가자 × 1만원 (싱글렛 여부 무관, 1인 1만원 고정)

**예시 (6월 기준)**: 5월 11명, 6월 12명 활성
→ 보증금: 11만 + 12만 = 23만원
→ 참가비: 12 × 1만 = 12만원
→ **총 참가비 = 35만원**

### 8.2 총 회식비

```
총 회식비 = 총 참가비 - 총 환급금
```

### 8.3 1인당 회식비 상한 (참여 개월 수 비례 지분제)

**전체 참여자** 대상으로 지분 계산 (프로젝트 시작월부터 현재월까지의 참여 개월 수 기준):

```
1인 지분 = 본인 참여 개월 수 (프로젝트 시작월부터)
1인 상한 = 총 회식비 × (본인 지분 / 전체 참여자 지분 합)
```

**예시**: 풀 50만원, A(5개월), B(5개월, 불참), C(3개월) 회식 참석
→ 참석자 지분: A(5) + C(3) = 8
→ A 상한: 50만 × 5/8 = 312,500원
→ C 상한: 50만 × 3/8 = 187,500원

### 8.4 규칙

- 회식비는 회식에 참여했을 때 회식비로만 지원됨. 다른 방법으로 받거나 환급될 수 없음.
- 상한 초과분은 자비 부담
- 회식비가 남는다면 **크루 회비로 귀속**

## 9. 크루 진행현황 그래프

### 9.1 축

- **X축**: 선택된 월의 1일 ~ 말일
- **Y축**: 마일리지(절대값) 또는 달성률(%)
- **토글 버튼**: "마일리지" / "달성률" (우측 상단)

### 9.2 데이터 표시

- **표시 대상**: 해당 월에 목표가 있거나 기록이 있는 참여자만
- **누적 마일리지**: 각 일자까지의 누적합
- **달성률**: `min(누적 마일리지 / 목표, 100)%` — 100%가 최대
- **목표 기준선**:
  - 마일리지 모드: 본인 목표값 수평 점선
  - 달성률 모드: 100% 수평 점선

### 9.3 툴팁

- 상위 5명만 표시 (값 내림차순)
- 본인이 상위 5명에 없으면 추가로 표시
- 나머지는 "외 N명"으로 축약

### 9.4 그래프 갱신

- 기록 입력/수정/삭제 후 `router.refresh()` + `refreshKey`로 즉시 반영

## 10. 랜덤 후기

- **위치**: 크루 진행현황 그래프와 N월 진행현황 통계 사이
- **로직**: 프로젝트 전체 기록 중 `review`가 비어있지 않은 최근 10개에서 랜덤 1개 선택
- **표시**: "이름 · 종목 거리km" + 후기 텍스트
- **후기가 없으면**: 아무것도 표시하지 않음

---

## 11. N월 진행현황 통계

그래프와 같은 border 블록 안에 위치. 그래프 모드(마일리지/달성률)에 연동. **3열 × 2행 그리드**.

**1행**:

| 참가 인원 | 달성 인원 | 평균 마일리지 / 평균 달성률 |
|:---:|:---:|:---:|
| N명 | N명 | N km / N% (모드 연동) |

**2행** (실전 기간만):

| 총 참가비 | 총 회식비 | 총 환급금 |
|:---:|:---:|:---:|
| N원 | N원 | N원 |

- **참가 인원**: 해당 월 이전에 참가 신청한 전원 (`start_month <= 해당 월`, 목표 유무 무관)
- **달성 인원**: 목표 달성한 인원 수 (단순 N명 표시)
- **총 참가비**: Σ(월별 활성인원 × 보증금) + 활성 참가자 × 참가비. 보증금은 프로젝트 시작월부터 카운트.
- **총 환급금**: 전월 + 당월 누적 환급금 합산
- **총 회식비**: 총 참가비 - 총 환급금
- **평균 달성률**: 각 참여자의 `min(달성/목표, 1)` 평균 × 100
- **목표 fallback**: `mileage_goal`이 없으면 `initial_goal` 사용
- **그래프 모드 공유**: ChartModeContext (React Context)

---

## 12. 내 현황

환급/회식지원비와 **하나의 카드로 통합**되어 표시.

### 12.1 상단: 진행 상태 (3열 그리드)

| 이번달 | 진행률 | 기간 대비 |
|:---:|:---:|:---:|
| 현재마일리지 / 목표 km | N% | N% |

- **이번달**: 현재 마일리지 / 목표 형태로 표시 (예: `42.5 / 100 km`)
- **진행률**: `min(현재 / 목표, 100)%`
- **기간 대비**: 진행률 / (경과일 / 총 일수)

### 12.2 프로그레스 바

- 진행률 시각화 (`h-2` 높이의 수평 바)

### 12.3 일일 필요 거리

- `(목표 - 현재) / 잔여일`
- 달성 시 "이번 달 목표 달성 완료!" 표시
- 월 종료 시 "이번 달이 종료되었습니다." 표시

### 12.4 하단: 환급/회식지원비 (실전 기간만, border-t 구분)

| 내 환급금 | 내 회식지원비 |
|:---:|:---:|
| N원 (누적) | N원 (예상) |

안내 문구:
- 환급금은 이벤트 종료 후 일괄 환급됩니다.
- 회식지원금은 예상금액으로 참여인원수에 따라 달라집니다.

## 13. 종목별 마일리지 (도넛 차트)

- **위치**: 내 현황과 내 기록 사이
- **구성**: 왼쪽 도넛 차트 + 오른쪽 범례
- **데이터**: 선택된 월의 본인 기록을 종목별로 집계
  - 종목별 **base_mileage** 기준으로 표시
  - 이벤트 보너스(`final_mileage - base_mileage`)는 별도 "이벤트 보너스"로 분리
- **범례**: 종목명 + 마일리지, 거리/고도 기여분 세부 표시
  - 예: `트레일러닝 76.5 km` → `거리 55.0 + 고도 21.5`
  - 수영은 거리만 표시 (고도 없음)
- **툴팁**: 호버/터치 시 종목명, 마일리지, 거리+고도 내역, 비율(%) 표시
- **색상**: 러닝(파랑), 트레일러닝(초록), 자전거(주황), 수영(시안), 이벤트(보라)
- **기록 없으면**: 컴포넌트 미표시

## 14. 기록 입력

### 14.1 입력 항목

| 항목 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 날짜 | Date | O | 기본값: 오늘 |
| 종목 | Select | O | 러닝 / 트레일러닝 / 자전거 / 수영 |
| 거리 | Number (km) | O | |
| 상승고도 | Number (m) | - | 수영 제외 전 종목, 선택 입력 |
| 이벤트 | Checkbox | - | 활성 이벤트 목록에서 복수 선택 |
| 후기 | Text | - | 한 줄 텍스트, 선택 |

### 14.2 입력 제한

- **미래 날짜**: 입력 불가
- **당월 기록**: 자유 입력
- **전월 기록**: 매월 3일까지만 입력 가능. 이후는 관리자만 가능.
  - 예: 6월 4일 → 5월 기록 입력 불가 (관리자 제외)

### 14.3 수정/삭제

- 본인 기록만 수정/삭제 가능
- 수정: Sheet(하단 시트)에 ActivityLogForm edit 모드
- 삭제: AlertDialog 확인 후 삭제
- 수정/삭제에도 입력과 동일한 시간 제한 적용 (전월 기록은 3일까지만, 이후 관리자만)
- 전월 기록 잠금 시 UI에서 버튼 비활성 + 안내 메시지

### 14.4 내 기록 목록

- **월별 필터**: 선택된 월의 기록만 표시
- **정렬**: 날짜 내림차순 (최신순)
- **초기 표시**: 5개, "더보기" 버튼으로 5개씩 추가
- **즉시 반영**: 입력/수정/삭제 후 `router.refresh()`

## 15. 마일리지런 규칙 보기

### 15.1 하단 버튼

- **위치**: 페이지 최하단 (참여자/비참여자 모두에게 표시)
- **형태**: dashed border outline 버튼 "마일리지런 규칙 보기"
- **동작**: 하단 Sheet로 규칙 전문 표시

### 15.2 규칙 내용 (공통 컴포넌트)

참가 신청 페이지 상단과 규칙 Sheet에서 동일한 내용을 공유:

1. 종목별 마일리지 환산 (좌우 정렬 테이블)
2. 이벤트 배율
3. 월 목표 자동 상향 (좌우 정렬 테이블)
4. 보증금 & 환급
5. 참가비 & 회식비: 참가비 20,000원 (싱글렛 보유 시 10,000원). 미환급 보증금 + 1만원이 회식비 풀로 적립되며, 참여 개월 수에 비례하여 배분.
6. 기록 수정 규칙

### 15.3 하단 안내문구

- 마일리지런은 중도 포기가 없습니다. 부상, 이사 등 어떤 사유에도 중도 포기되지 않습니다.
- 회식비는 회식에 참여했을 때 회식비로만 지원됩니다. 다른 방법으로 받거나 환급될 수 없습니다.
- 회식비가 남는다면 회비로 귀속됩니다.

## 부록: 계산 예시

### 예시 1: 기본 기록

> 러닝 10km, 상승고도 200m, 비 올 때(×1.2)
> 기본: 10 + 2 = 12 마일리지
> 최종: 12 × 1.2 = **14.4 마일리지**

### 예시 2: 복합 이벤트

> 트레일러닝 15km, 상승고도 500m, 비(×1.2) + 모임(×1.1)
> 기본: 15 + 5 = 20 마일리지
> 최종: 20 × 1.2 × 1.1 = **26.4 마일리지**

### 예시 3: 환급 계산

> 5월 시작, 초보(50), 싱글렛 보유
> 납부: 5만(보증금) + 1만(참가비) = 6만원
> 결과: 5월 100% / 6월 100% / 7월 100% / 8월 80% / 9월 40%
> 환급: (1.0+1.0+1.0+0.8+0.4) × 1만 = **4.2만원**
> 미환급: 0.8만원 → 회식비 풀

### 예시 4: 회식비 배분

> 풀: 30만원
> 회식 참석자: A(5개월), B(4개월), C(3개월)
> 총 지분: 12
> A: 30만 × 5/12 = 125,000원
> B: 30만 × 4/12 = 100,000원
> C: 30만 × 3/12 = 75,000원

### 예시 5: 총 참가비 계산 (6월 기준)

> 5월 활성 11명, 6월 활성 12명 (1명 중도 합류)
> 보증금: 11 × 1만 + 12 × 1만 = 23만원
> 참가비: 12 × 1만 = 12만원
> **총 참가비: 35만원**
> 총 환급금: 5월 누적 + 6월 누적
> 총 회식비: 35만 - 총 환급금
