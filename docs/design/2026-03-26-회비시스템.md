## 1. 개요

기강 크루의 월 회비 납부·관리 시스템입니다.
카카오뱅크 계좌와 서버를 직접 연동할 수 없으므로, 모임장이 CSV를 주기적으로 업로드하면 서버가 이를 파싱하여 회원별 납부 내역을 관리합니다.

## 2. 회비 정책

### 기본 정책

| 항목 | 내용 |
|------|------|
| 월 회비 | 2,000원 (변경 시 `dues_config` 테이블에 이력 관리) |
| 납부 수단 | 카카오뱅크 계좌 입금 (3333-09-6788223) |
| 입금자명 | 본인 실명 (띄어쓰기 없이) |
| 회비 환불 | 불가 |
| 예치금 환불 | 가능 (관리자 처리) |
| 내역 공개 | 비공개 (관리자만 전체 열람 가능) |
| 납부 마감일 | 매월 말일 (미결정 — 9. 미결정 사항 참조) |

### 면제 정책

아래 상황에서 모임장이 개별 회원에 대해 특정 월 회비를 면제할 수 있습니다.

- 타지역 이사로 활동 불가
- 장기 부상·휴식
- 기타 모임장 판단

면제 시 **면제 사유, 면제 기간, 승인한 관리자 ID**를 반드시 기록합니다. (`member_due_records.exemption_reason`, `exempted_by` 컬럼)

### 신규 가입자 회비 시작 시점

- 월 중간(1~14일) 가입: 해당 월 회비 부과
- 월 중간(15일 이후) 가입: 해당 월 회비 면제 처리 (`exempted` 상태, 자동 면제)
- 시작 기준은 온보딩 완료(`members.status = 'active'` 전환) 시점

> 정확한 기준일은 미결정 사항 참조.

### 탈퇴 시 예치금 처리

탈퇴 플로우 연계 필요:
1. 탈퇴 요청 접수 시 미납금·예치금 잔액 안내
2. 예치금 잔액 > 0이면 환불 신청 또는 포기 선택
3. 처리 완료 후 탈퇴 확정

> 탈퇴 플로우 설계는 별도 문서에서 정의 예정.

### 예치금 시스템

회비 초과 납부 시 초과금은 예치금으로 적립되며, 이후 월 회비에 자동 차감됩니다.

**예시**: 1월에 12,000원 납부

| 월 | 처리 내용 | 예치금 잔액 |
|----|-----------|------------|
| 1월 | 2,000원 → 회비 / 10,000원 → 예치금 적립 | 10,000원 |
| 2월 | 예치금에서 2,000원 차감 | 8,000원 |
| 3월 | 예치금에서 2,000원 차감 | 6,000원 |
| ... | ... | ... |
| 6월 | 예치금에서 2,000원 차감 | 0원 |
| 7월 | 예치금 소진 → 미납 발생 | 0원 |

회원 프로필에서 예치금 차감 시 "예치금 자동 차감 (차감액: 2,000원, 잔액: N원)" 형식으로 표시합니다.

### 회비 사용 원칙

- 모임 행사·이벤트 비용
- 운동 동기부여 관련 지출
- 모임 발전을 위한 활동
- **사용 권한**: 모임장 단독 판단 또는 운영진 건의

## 3. 데이터 수집 방식 (CSV 업로드)

### 흐름

```
카카오뱅크 앱 → 거래 내역 CSV 다운로드 (권장: 매월 초 전월 내역 다운로드)
→ 모임장(관리자) 업로드
→ 서버 파싱 및 회원 자동 매칭
→ 관리자 검토 및 수동 보정
→ 회원 회비 상태 반영
→ 월 마감 실행
```

### CSV 업로드 권장 주기

매월 초(1~3일) 전월 거래 내역을 업로드합니다. 관리자 대시보드에 마지막 업로드 일시를 표시하여 업로드 누락을 방지합니다.

### CSV 포맷 (카카오뱅크 입출금 내역)

카카오뱅크 거래내역 CSV 예상 컬럼:

| 컬럼 | 예시 | 비고 |
|------|------|------|
| 거래일 | 2026.03.05 | |
| 거래시간 | 14:32:00 | |
| 거래유형 | 입금 / 출금 | |
| 거래금액 | 2000 | |
| 잔액 | 45000 | |
| 내용 | 홍길동 | 입금자명 또는 메모 |

> 실제 컬럼명은 카카오뱅크 CSV 샘플 확인 후 파서 구현 시 맞춰야 합니다.
> 카카오뱅크 CSV는 EUC-KR 인코딩일 수 있어 서버에서 변환 처리 필요.

### 회비 명목 판단 기준

CSV 한 건이 회비 납부인지, 지출인지, 기타 입금인지 구분이 필요합니다.

| 조건 | 분류 | 비고 |
|------|------|------|
| 거래유형 = 입금 + 금액이 2,000원 단위 + 이름 매칭 가능 | 회비 or 예치금 | 자동 분류 후 관리자 확인 |
| 거래유형 = 출금 | 지출 | 회식비, 행사비 등 |
| 거래유형 = 입금 + 이름 매칭 불가 | 미확인 | 관리자 수동 처리 |

> 최종 카테고리 지정은 관리자가 수동 확인·보정합니다.

### 중복 업로드 방지

- 파일 내용의 SHA-256 해시(`content_hash`)를 `csv_uploads`에 저장
- 동일 해시 파일 재업로드 시 서버에서 오류 반환, 프론트에서 "이미 업로드된 파일입니다" 안내
- 잘못 업로드된 CSV는 `delete-csv-upload` 액션으로 관련 `dues_transactions` cascade delete 후 롤백 가능 (단, 이미 `member_due_records`에 반영된 경우 관리자 수동 확인 필요)

## 4. 회원 매칭 로직

### 자동 매칭

1. CSV 내 `내용` 필드에서 이름 추출
2. 현재 `active` 상태 회원 목록과 비교 (이름 기준, 완전 일치)
3. **정확히 1명 매칭** → `matched` 상태로 자동 연결
4. **0명 또는 2명 이상 매칭** → `unmatched` 또는 `ambiguous` 상태로 관리자 대기열에 올림

이름 정규화: 공백 제거 후 비교. 포함 검색은 사용하지 않고 완전 일치만 적용합니다.

### 동명이인 처리 (`ambiguous`)

자동 매칭하지 않고 `ambiguous` 상태로 남겨 관리자가 직접 지정합니다.
관리자 미매칭 처리 화면에서 후보 회원 목록(이름 + 전화번호 뒷자리 힌트)을 표시하여 선택할 수 있도록 합니다.

### 미매칭 거래 처리 (`unmatched`)

- 회비 관련이 아닌 것으로 판단되는 경우: `other` 카테고리로 분류, 관리자만 열람 가능
- 회비일 가능성이 있는 경우: 관리자가 전체 회원 검색 후 수동으로 회원 UUID 지정

> `ambiguous`(동명이인 후보 선택)와 `unmatched`(전체 검색 후 지정)는 화면에서 별도 UI로 처리합니다.

## 5. DB 스키마

### `dues_config` — 회비 금액 이력 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| effective_from | date | 해당 금액 적용 시작 월 (YYYY-MM-01) |
| amount | int | 월 회비 금액 (원) |
| created_by | uuid FK → members | 변경한 관리자 |
| created_at | timestamptz | |

> `close-month` 실행 시 해당 월 기준 `dues_config`에서 금액을 조회하여 하드코딩을 피합니다.

### `dues_expense_records` — 회비 지출 기록 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| transaction_id | uuid FK → dues_transactions (nullable) | CSV에서 파싱된 지출 거래 |
| expense_date | date | 지출일 |
| amount | int | 지출 금액 (양수) |
| title | text | 지출 항목명 (예: 2월 뒤풀이) |
| description | text (nullable) | 상세 내용 |
| created_by | uuid FK → members | 등록한 관리자 |
| created_at | timestamptz | |

### `dues_month_close_log` — 월 마감 이력 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| year_month | text PK | `YYYY-MM` 형식, 월당 1건 |
| closed_at | timestamptz | 마감 실행 시각 |
| closed_by | uuid FK → members | 마감 실행 관리자 |

> 이 테이블로 `close-month` 중복 실행을 방지합니다. 이미 레코드가 있는 월은 재마감 불가.
> 마감 후에는 해당 월 데이터 수정이 잠기며, 수정이 필요하면 관리자가 마감을 해제(레코드 삭제) 후 재처리합니다.

| 컬럼 | 타입 | 설명 |

| 컬럼 | 타입 | 설명 |

| id | uuid PK | |

| transaction_time | time | 거래시간 |

## 6. 백엔드 기능 명세

### 서버 액션 / API

| 기능 | 경로 | 설명 |
|------|------|------|
| CSV 업로드 및 파싱 | `actions/dues/upload-csv.ts` | 파일 수신 → SHA-256 중복 확인 → EUC-KR 변환 → 파싱 → dues_transactions 저장 → 자동 매칭 실행 |
| CSV 롤백 | `actions/dues/delete-csv-upload.ts` | 업로드 및 관련 dues_transactions cascade delete (member_due_records 반영 전 단계만 가능) |
| 회원 수동 매칭 | `actions/dues/match-transaction.ts` | 관리자가 미매칭 거래에 회원 UUID 지정. 이미 matched 상태인 건 덮어쓰기 가능 (확인 다이얼로그 필요) |
| 카테고리 변경 | `actions/dues/update-category.ts` | 거래 카테고리 수정 |
| 면제 처리 | `actions/dues/exempt-member.ts` | 특정 회원·특정 월 회비 면제. 사유·승인자 기록 필수 |
| 예치금 환불 | `actions/dues/refund-deposit.ts` | 예치금 차감 + deposit_transactions 기록. SELECT FOR UPDATE RPC 사용 |
| 예치금 충전 (수동) | `actions/dues/charge-deposit.ts` | 예치금 직접 충전 (CSV 매칭 외 수동 보정용) |
| 월별 회비 마감 | `actions/dues/close-month.ts` | dues_month_close_log 존재 확인 → 예치금 자동 차감 → 미납 확정 → 마감 로그 INSERT |
| 지출 기록 등록 | `actions/dues/create-expense.ts` | dues_expense_records 등록 |

### 자동 매칭 로직 (upload-csv 내부)

```
CSV 행 순회
  → 파싱 오류 시 parse_error 컬럼 기록, 해당 행 skip (전체 롤백하지 않음)
  → 이름 공백 제거 후 active 회원 검색 (완전 일치)
  → 매칭 1명: matched_member_id 설정, match_status = 'matched'
  → 매칭 0명: match_status = 'unmatched'
  → 매칭 2명 이상: match_status = 'ambiguous'
```

> 파싱 중 오류가 발생한 행은 `parse_error`에 기록하고 건너뜁니다. 업로드 결과 요약에 "파싱 오류 N건"을 표시합니다.

### 월별 회비 마감 로직 (close-month)

```
dues_month_close_log에 해당 year_month 존재 확인
  → 존재하면: 오류 반환 "이미 마감된 월입니다"
  → 없으면:
      dues_config에서 해당 월 회비 금액 조회
      해당 월 미납 회원 조회 (member_due_records.status = 'unpaid' or 레코드 없음)
        → 예치금 잔액 >= 회비 금액인 회원:
            SELECT FOR UPDATE (RPC) → balance -= 회비금액
            status = 'covered_by_deposit'
            deposit_transactions 차감 기록 (member_due_record_id 연결)
        → 예치금 없는 회원: status = 'unpaid' 확정
      dues_month_close_log INSERT (closed_by, closed_at)
```

### 관리자 판별 (RLS)

`members` 테이블의 `admin boolean` 컬럼 기준:

```sql
CREATE POLICY "관리자 전체 조회"
ON dues_transactions FOR SELECT
USING (
  EXISTS (
    SELECT 1 FROM members
    WHERE members.id = auth.uid()
    AND members.admin = true
  )
);
```

## 7. 프론트엔드 기능 명세

### 라우팅 구조

관리자 전용 페이지는 별도 Route Group `(admin)`으로 분리합니다.

```
app/
  (admin)/
    layout.tsx          # 관리자 레이아웃 (관리자 여부 서버 검증 + redirect)
    dues/
      page.tsx          # 관리자 대시보드 (요약 수치)
      transactions/
        page.tsx        # 거래 내역 조회 + 미매칭 처리
      members/
        page.tsx        # 회원별 회비 현황
      close/
        page.tsx        # 월 마감
      expenses/
        page.tsx        # 지출 기록 관리
```

관리자 레이아웃에서 `members.admin` 확인 후 비관리자는 홈으로 redirect합니다.

### 회원 프로필 — 회비 섹션

회원 본인만 열람 가능.

| 표시 항목 | 설명 |
|-----------|------|
| 최근 납부일 | 가장 최근 `paid` or `covered_by_deposit` 기록의 거래일 |
| 예치금 잔액 | `member_deposits.balance` |
| 예치금 변동 이력 | `deposit_transactions` 최근 N건 (`display_reason` 표시) |
| 미납금 | `unpaid` 상태 레코드 수 × 회비금액, 미납 월 목록 표시 |
| 월별 납부 내역 | 최근 12개월 상태 목록 (더보기로 전체 이력 조회 가능) |
| 면제 현황 | `exempted` 상태 월 + `display_reason` (왜 면제인지, 언제까지인지) |
| 납부 안내 | 계좌번호 (3333-09-6788223), 입금자명 작성법 안내 |
| 마지막 업로드 시각 | CSV 마지막 업로드 시각 표시 (내역 반영 기준 안내) |

**상태별 표시 예시**

- `covered_by_deposit`: "예치금 자동 차감 (차감액: 2,000원, 잔액: N원)"
- `exempted`: "면제 처리됨 — 사유: 장기 부상"
- `unpaid`: "미납 — 납부 방법 안내 보기" (링크 또는 모달)

**빈 상태**: 납부 이력이 없을 때 "아직 납부 이력이 없습니다" 안내 문구 표시

### 관리자 대시보드 — 요약 수치

| 표시 항목 | 설명 |
|-----------|------|
| 이번 달 납부 완료 | 납부 완료 인원 / 전체 인원 |
| 이번 달 미납 | 미납 인원 수 |
| 총 예치금 잔액 | 전체 회원 예치금 합계 |
| 마지막 CSV 업로드 | 업로드 일시 |
| 미매칭 건수 | 처리 대기 중인 미매칭 거래 수 |

### 관리자 페이지 — 거래 내역 조회 및 미매칭 처리

| 기능 | 설명 |
|------|------|
| CSV 업로드 | 파일 선택 버튼 + 드래그앤드롭 지원. 업로드 중 스피너 표시. 완료 후 파싱 결과 요약(총 N건, 매칭 N건, 미매칭 N건, 오류 N건) 표시. 확정 전 취소 가능. |
| 거래 내역 조회 | 전체 거래 목록. 카테고리·매칭 상태 필터는 URL 쿼리 파라미터로 관리(`?category=due&status=unmatched`). |
| 미매칭 처리 (`unmatched`) | 전체 회원 검색 후 수동 지정. |
| 동명이인 처리 (`ambiguous`) | 후보 회원 목록(이름 + 전화번호 뒷자리 힌트) 표시 후 선택. |
| 카테고리 변경 | 거래 카테고리 인라인 수정. |

### 관리자 페이지 — 회원별 회비 현황

| 기능 | 설명 |
|------|------|
| 회원별 납부 상태 | 멤버 목록 + 이번 달 납부 상태 한눈에 보기 |
| 면제 처리 | 회원 + 연월 선택 + 사유 입력 후 면제 적용 |
| 예치금 환불 | 회원 선택 + 금액 입력 (잔액 초과 입력 불가) |
| 미납자 목록 복사 | 미납자 이름 목록 클립보드 복사 버튼 |
| 미납 알림 문자 템플릿 | 미납 안내 문자 템플릿 생성 (이름 치환 포함) |

### 관리자 페이지 — 월 마감

| 기능 | 설명 |
|------|------|
| 마감 실행 | 대상 월 선택 → 예치금 자동 차감 + 미납 확정. 이미 마감된 월은 버튼 비활성화. 실행 전 확인 다이얼로그 표시. |

| 마감 해제 | 마감된 월 해제 (재처리용). 이미 반영된 예치금 차감 역산은 수동 처리 필요 안내. |

## 8. RLS (Row Level Security) 방침

관리자 판별: `members.admin = true` 조건 기준.

| 테이블 | 본인 조회 | 관리자 전체 조회 | 관리자 수정 |
|--------|----------|----------------|------------|
| dues_transactions | ✗ | ✅ | ✅ |
| member_due_records | ✅ (본인 것만) | ✅ | ✅ |
| member_deposits | ✅ (본인 것만) | ✅ | ✅ |
| deposit_transactions | ✅ (본인 것만, display_reason만 표시) | ✅ (전체 컬럼) | ✅ |
| csv_uploads | ✗ | ✅ | ✅ |
| dues_config | ✗ | ✅ | ✅ |
| dues_expense_records | ✗ | ✅ | ✅ |
| dues_month_close_log | ✗ | ✅ | ✅ |

> 일반 회원이 자신의 납부일(`dues_transactions.transaction_date`)을 확인해야 할 때는 Supabase RPC(PostgreSQL Function)를 통해 필요한 컬럼만 반환합니다. 일반 회원에게 직접 `dues_transactions` 조회 권한을 부여하지 않습니다.

## 9. 미결정 사항

| 항목 | 내용 | 우선순위 |
|------|------|---------|
| CSV 컬럼 정확한 포맷 | 카카오뱅크 실제 CSV 샘플 필요 | 높음 |
| 월 마감 시점 | 수동 실행 vs 자동 스케줄 (매월 1일 등). 자동이면 `pg_cron` 또는 외부 cron + API 라우트 방식 선택 필요 | 중간 |
| 신규 가입자 회비 시작 기준일 | 월 중간 가입 시 15일 기준 면제 여부 확정 필요 | 중간 |
| 납부 마감일 | 매월 몇 일까지로 정의할지 | 중간 |
| 알림 구현 여부 | 납부 확인·미납 알림 인앱 알림 또는 푸시 구현 여부 | 중간 |
| 예치금 환불 프로세스 | 환불 요청 → 모임장 승인 → 처리 or 즉시 처리 | 중간 |
| 잘못 매칭 이의 제기 | 회원이 잘못된 매칭을 발견했을 때 문의 경로 (별도 문의 기능 vs 카카오 채팅) | 낮음 |
| 탈퇴 시 예치금 처리 세부 절차 | 탈퇴 플로우 문서 연계 | 낮음 |
| 관리자 대시보드 회비 사용 내역 공개 여부 | 지출 내역을 회원에게 공개할지 관리자만 열람할지 | 낮음 |
