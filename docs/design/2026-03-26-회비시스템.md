# 회비 시스템 설계

## 1. 개요

기강 크루의 월 회비 납부·관리 시스템입니다.
카카오뱅크 계좌와 서버를 직접 연동할 수 없으므로, 모임장이 CSV를 주기적으로 업로드하면 서버가 이를 파싱하여 회원별 납부 내역을 관리합니다.

---

## 2. 회비 정책

### 기본 정책

| 항목 | 내용 |
|------|------|
| 월 회비 | 2,000원 |
| 납부 수단 | 카카오뱅크 계좌 입금 (3333-09-6788223) |
| 회비 환불 | 불가 |
| 예치금 환불 | 가능 |
| 내역 공개 | 비공개 (관리자만 열람 가능) |

### 면제 정책

아래 상황에서 모임장이 개별 회원에 대해 특정 월 회비를 면제할 수 있습니다.

- 타지역 이사로 활동 불가
- 장기 부상·휴식
- 기타 모임장 판단

### 예치금 시스템

회비 초과 납부 시 초과금은 예치금으로 적립되며, 이후 월 회비에 자동 차감됩니다.

**예시**: 1월에 12,000원 납부

| 월 | 처리 내용 | 예치금 잔액 |
|----|-----------|------------|
| 1월 | 2,000원 → 회비 / 10,000원 → 예치금 적립 | 10,000원 |
| 2월 | 예치금에서 2,000원 차감 | 8,000원 |
| 3월 | 예치금에서 2,000원 차감 | 6,000원 |
| ... | ... | ... |
| 6월 | 예치금에서 2,000원 차감 | 0원 |
| 7월 | 예치금 소진 → 미납 발생 | 0원 |

### 회비 사용 원칙

- 모임 행사·이벤트 비용
- 운동 동기부여 관련 지출
- 모임 발전을 위한 활동
- **사용 권한**: 모임장 단독 판단 또는 운영진 건의

## 3. 데이터 수집 방식 (CSV 업로드)

### 흐름

```
카카오뱅크 앱 → 거래 내역 CSV 다운로드
→ 모임장(관리자) 업로드
→ 서버 파싱 및 회원 자동 매칭
→ 관리자 검토 및 수동 보정
→ 회원 회비 상태 반영
```

### CSV 포맷 (카카오뱅크 입출금 내역)

카카오뱅크 거래내역 CSV 예상 컬럼:

| 컬럼 | 예시 | 비고 |
|------|------|------|
| 거래일 | 2026.03.05 | |
| 거래시간 | 14:32:00 | |
| 거래유형 | 입금 / 출금 | |
| 거래금액 | 2000 | |
| 잔액 | 45000 | |
| 내용 | 홍길동 | 입금자명 또는 메모 |

> 실제 컬럼명은 카카오뱅크 CSV 샘플 확인 후 파서 구현 시 맞춰야 합니다.

### 회비 명목 판단 기준

CSV 한 건이 회비 납부인지, 지출인지, 기타 입금인지 구분이 필요합니다.

| 조건 | 분류 | 비고 |
|------|------|------|
| 거래유형 = 입금 + 금액이 2,000원 단위 + 이름 매칭 가능 | 회비 or 예치금 | 자동 분류 후 관리자 확인 |
| 거래유형 = 출금 | 지출 | 회식비, 행사비 등 |
| 거래유형 = 입금 + 이름 매칭 불가 | 미확인 | 관리자 수동 처리 |

> 최종 카테고리 지정은 관리자가 수동 확인·보정합니다.

## 4. 회원 매칭 로직

### 자동 매칭

1. CSV 내 `내용` 필드에서 이름 추출
2. 현재 활성화된 회원 목록과 비교 (이름 기준)
3. **정확히 1명 매칭** → `matched` 상태로 자동 연결
4. **0명 또는 2명 이상 매칭** → `unmatched` 상태로 관리자 대기열에 올림

### 동명이인 처리

자동 매칭하지 않고 `unmatched` 상태로 남겨 관리자가 직접 지정합니다.

### 미매칭 거래 처리

- 회비 관련이 아닌 것으로 판단되는 경우: `other` 카테고리로 분류, 관리자만 열람 가능
- 회비일 가능성이 있는 경우: 관리자가 수동으로 회원 UUID 지정

## 5. DB 스키마

### `csv_uploads` — CSV 업로드 이력

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| uploaded_by | uuid FK → members | 업로드한 관리자 |
| file_name | text | 원본 파일명 |
| row_count | int | 파싱된 행 수 |
| uploaded_at | timestamptz | 업로드 시각 |

### `dues_transactions` — CSV 원시 거래 내역

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| csv_upload_id | uuid FK → csv_uploads | 출처 업로드 |
| transaction_date | date | 거래일 |
| transaction_time | time | 거래시간 |
| amount | int | 금액 (양수=입금, 음수=출금) |
| raw_name | text | CSV 내 이름/메모 원문 |
| category | enum | `due` / `expense` / `other` |
| matched_member_id | uuid FK → members (nullable) | 매칭된 회원 |
| match_status | enum | `matched` / `unmatched` / `ambiguous` |
| created_at | timestamptz | |

**category 값 설명**

| 값 | 의미 |
|----|------|
| `due` | 회비 or 예치금 납부 |
| `expense` | 지출 (회식비, 행사비 등) |
| `other` | 기타 입금 (매칭 불필요) |

### `member_due_records` — 회원별 월 회비 납부 기록

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| member_id | uuid FK → members | |
| year_month | text | `YYYY-MM` 형식 |
| status | enum | `paid` / `unpaid` / `exempted` / `covered_by_deposit` |
| transaction_id | uuid FK → dues_transactions (nullable) | 연결된 거래 |
| created_at | timestamptz | |

**status 값 설명**

| 값 | 의미 |
|----|------|
| `paid` | 직접 납부 |
| `unpaid` | 미납 |
| `exempted` | 모임장 면제 |
| `covered_by_deposit` | 예치금으로 차감 처리 |

### `member_deposits` — 회원별 예치금 잔액

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| member_id | uuid FK → members UNIQUE | |
| balance | int | 현재 예치금 잔액 (원) |
| updated_at | timestamptz | |

### `deposit_transactions` — 예치금 변동 내역

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| member_id | uuid FK → members | |
| amount | int | 양수=충전, 음수=차감·환불 |
| type | enum | `charge` / `deduction` / `refund` |
| year_month | text (nullable) | 회비 차감 시 해당 월 |
| reason | text | 비고 |
| created_at | timestamptz | |

## 6. 백엔드 기능 명세

### 서버 액션 / API

| 기능 | 경로 | 설명 |
|------|------|------|
| CSV 업로드 및 파싱 | `actions/dues/upload-csv.ts` | 파일 수신 → 파싱 → dues_transactions 저장 → 자동 매칭 실행 |
| 회원 수동 매칭 | `actions/dues/match-transaction.ts` | 관리자가 미매칭 거래에 회원 UUID 지정 |
| 카테고리 변경 | `actions/dues/update-category.ts` | 거래 카테고리 수정 |
| 면제 처리 | `actions/dues/exempt-member.ts` | 특정 회원·특정 월 회비 면제 |
| 예치금 환불 | `actions/dues/refund-deposit.ts` | 예치금 차감 + deposit_transactions 기록 |
| 월별 회비 마감 | `actions/dues/close-month.ts` | 특정 월 마감 시 예치금 자동 차감, 미납 확정 |

### 자동 매칭 로직 (upload-csv 내부)

```
CSV 행 순회
  → 이름으로 활성 회원 검색
  → 매칭 1명: matched_member_id 설정, match_status = 'matched'
  → 매칭 0명 or 2명 이상: match_status = 'unmatched' or 'ambiguous'
```

### 월별 회비 마감 로직 (close-month)

```
해당 월 미납 회원 조회
  → 예치금 잔액 > 0인 회원: balance -= 2000, status = 'covered_by_deposit'
  → 예치금 없는 회원: status = 'unpaid' 확정
```

## 7. 프론트엔드 기능 명세

### 회원 프로필 — 회비 섹션

회원 본인만 열람 가능.

| 표시 항목 | 설명 |
|-----------|------|
| 최근 납부일 | 가장 최근 `paid` or `covered_by_deposit` 기록의 거래일 |
| 예치금 잔액 | `member_deposits.balance` |
| 미납금 | `unpaid` 상태 레코드 수 × 2,000원 |
| 월별 납부 내역 | 최근 N개월 상태 목록 (`paid` / `unpaid` / `exempted` 등) |

### 관리자 페이지 — 회비 관리

관리자(`admin = true`)만 접근 가능.

| 기능 | 설명 |
|------|------|
| CSV 업로드 | 파일 선택 → 업로드 → 파싱 결과 미리보기 |
| 거래 내역 조회 | 전체 거래 목록, 카테고리·매칭 상태 필터 |
| 미매칭 처리 | 미매칭 거래 목록에서 회원 수동 지정 |
| 회원별 회비 현황 | 멤버 목록 + 납부 상태 한눈에 보기 |
| 면제 처리 | 회원 + 연월 선택 후 면제 적용 |
| 예치금 환불 | 회원 선택 + 금액 입력 |
| 월 마감 | 특정 월 회비 마감 실행 |

|--------|----------|----------------|------------|
| dues_transactions | ✗ | ✅ | ✅ |
| member_due_records | ✅ (본인 것만) | ✅ | ✅ |
| member_deposits | ✅ (본인 것만) | ✅ | ✅ |
| deposit_transactions | ✅ (본인 것만) | ✅ | ✅ |
| csv_uploads | ✗ | ✅ | ✅ |

## 9. 미결정 사항

| 항목 | 내용 |
|------|------|
| CSV 컬럼 정확한 포맷 | 카카오뱅크 실제 CSV 샘플 필요 |
| 월 마감 시점 | 수동 실행 vs 자동 스케줄 (매월 1일 등) |
| 미납 알림 | 미납 회원에게 알림 전송 여부 |
| 예치금 환불 프로세스 | 환불 요청 → 모임장 승인 → 처리 or 즉시 처리 |
| 회비 금액 변경 | 향후 금액 변경 시 이력 관리 방식 |
