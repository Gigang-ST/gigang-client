# QUALITY-BAR — "기강에서 완성됐다"의 정의

> 품질 스윕 채점 루브릭. **살아있는 문서**: 스윕에서 "기준에 없는데 명백히 어색"이 나오면
> 항목을 추가하고 백로그에 적재한다. 채점: 항목별 **PASS / WARN(마찰) / FAIL(결함)**.
> FAIL→P0~P1, WARN→P1~P2. 프로파일: **web** (Next.js + 모바일 PWA 우선).

## 1. 첫인상
- [ ] 라우트에 로딩 스켈레톤(실제 레이아웃 치수 미러링, CLS 방지) — 첫 방문 blank 금지
- [ ] 데이터 0 빈 상태: 아이콘+안내+가능하면 CTA — 죽은 화면 금지 (`EmptyState` 사용)
- [ ] 신규 유저 첫 화면이 "다음에 뭘 하면 되는지" 안내
- [ ] 무거운 데이터는 Suspense/지연 분리 — 셸 먼저
- [ ] 첫 가치 도달까지 이탈 유발 지연 없음(백지·무한 스피너 금지)

## 2. 신뢰 (데이터 정합)
- [ ] 라벨과 데이터 의미 일치("오늘"엔 오늘 것만, "최근"은 정의된 최근)
- [ ] 시각·날짜 KST 고정(`lib/dayjs`만 사용), 형식 일관
- [ ] 귀속 정확: 작성자·담당자·팀 스코프 누수 없음
- [ ] 에러 상태: 네트워크·권한·백엔드 실패 시 사람말 안내(raw 에러·백지 금지)
- [ ] 쓰기 경로 멱등(중복 생성 금지), 카운트·배지 숫자 정확
- [ ] **되돌리기·감사가능성**: 데이터를 바꾸는 확정성 액션은 (a) 결과를 나중에 조회할 수 있고 (b) 오류 발견 시 정정 경로가 있다 — 둘 다 없으면 FAIL
- [ ] 부분 실패 시 정합성: 배치 쓰기가 중간에 실패해도 반쯤 반영된 상태로 남지 않는다(보상/트랜잭션)

## 3. 모바일
- [ ] 모든 콘텐츠 reflow — hidden으로 통째 증발 금지
- [ ] 터치 타깃 충분(≈44px), 하단 탭바 셸 정상
- [ ] 가로 스크롤·overflow 깨짐 없음(긴 텍스트 포함; 데이터 테이블은 명시적 스크롤 컨테이너만 허용)
- [ ] 인앱브라우저(카톡 등) 함정 안내

## 4. 인터랙션
- [ ] 액션 요소에 촉각 피드백(press/hover), 정적 요소엔 없음
- [ ] 지연 작업에 진행 표시 — 눌렀는데 무반응 금지
- [ ] prefers-reduced-motion 가드
- [ ] 입력 규약: 포커스·Enter 규약·연타 재진입 방지
- [ ] 대규모·비가역 액션은 실행 전 범위 고지(몇 건에 무슨 일이 일어나는지)

## 5. 카피·톤
- [ ] 제품 톤 일관(사람말투·간결·한국어)
- [ ] 사용자 화면에 영어 raw·개발 용어 노출 금지
- [ ] 아이콘/이모지 규율(DESIGN.md 준수)
- [ ] 라벨=실제 동작, 빈 상태 카피가 상황에 맞음(검색 무결과≠데이터 없음)

## 6. 이음새
- [ ] 화면 전환 시 맥락 유지(활성 필터·검색어)
- [ ] 화면 간 데이터 일치(관리자 화면↔회원 화면, 원장↔개인 내역)
- [ ] 비동기 작업 결과가 화면에 반영(새로고침 강요 금지)
- [ ] 딥링크·뒤로가기·새로고침에 상태 안 깨짐(로그인 왕복 포함)

## 재발 방지 게이트 (신규 기능 출고 시)
마무리 단계에서: ① 관련 카테고리 자가 채점 ② 영향받는 저니 1회 재워크. 반복 위반 패턴은 하드게이트로 승격.
