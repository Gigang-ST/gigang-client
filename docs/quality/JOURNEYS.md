## J1 — 신규 회원 첫인상

- **시작 조건**: 신규 계정(가입 이력 0) · 데이터 0 · **모바일 우선** (+카톡 인앱브라우저 케이스)
- **단계**: 1. 첫 방문(비로그인 홈) 2. 로그인(카카오/구글) 3. 가입 위저드(newbie→onboarding) 4. 홈 도달 — 다음 행동 안내 확인 5. PWA 설치·푸시 배너 동작
- **성공 기준**: 차단 없이 완주, 카테고리 FAIL 0
- **채점 화면**: `/`, `/auth/login`, `/newbie`, `/onboarding`

## J2 — 관리자 회비 거래처리 (업로드~확정~정정)

- **시작 조건**: 관리자 계정 · 은행 거래 엑셀 픽스처(회비/프로젝트성/지출/동명이인/신규입금자명 혼합) · 데스크톱 우선(운영 도구) + 모바일 1회
- **단계**:
  1. 은행 엑셀 업로드 → 버킷 요약(자동/확인필요/제외) 확인
  2. 인박스에서 확인필요 행 triage — 회원 매칭·분류·기억하기
  3. 일괄 확정 + 재계산 → 결과 메시지 확인
  4. 잔액 원장에서 반영 확인 (미납/예치 숫자)
  5. **오확정 발견 시나리오**: 어제 확정한 거래 1건이 잘못 매칭됐음을 발견 → 그 거래를 다시 찾고 → 정정 → 잔액 원상복구 확인
  6. 같은 파일 재업로드(중복 방지 확인)
- **성공 기준**: 6단계 전부 UI로 완주 가능 + 신뢰(2)·인터랙션(4) FAIL 0. 특히 5단계(감사·정정)가 존재해야 한다.
- **채점 화면**: `/admin/dues`, `/admin/dues/inbox`, (업로드 진입점 — 현재 소실), `/admin/dues/exemptions`
- **스윕 #1 노트**: 업로드 UI가 없어(QS-1) 화면 워크 불가 → 코드·git 기반 워크로 대체. QS-1 해소 후 실브라우저 재워크 필요.

## J3 — 회원 회비 확인

- **시작 조건**: 일반 회원 계정(납부 이력 있음 + 미납 상태 케이스) · 모바일
- **단계**: 1. 프로필→회비 내역 진입 2. 잔액·내역 이해(입금/면제/취소 라벨) 3. 이상 발견→"확인 요청" 전송 4. 관리자 측 수신 확인
- **성공 기준**: 회원이 자기 잔액의 근거를 스스로 이해할 수 있다
- **채점 화면**: `/profile/dues`

## J4 — 대회·기록

- **시작 조건**: 일반 회원 · 기록 0 상태와 기록 있음 상태 각 1회 · 모바일
- **단계**: 1. 대회 목록 탐색(기강/전체 탭) 2. 기록 등록 3. 랭킹 반영 확인 4. 프로필 기록 확인
- **채점 화면**: `/races`, `/records`, `/profile`

## J5 — 모임 만들기~참여

- **시작 조건**: 일반 회원 2계정(주최자/참여자) · 모바일
- **단계**: 1. 모임 생성 2. 다른 계정으로 참여 3. 알림 수신 4. 상세에서 참여자 확인
- **채점 화면**: `/gatherings`, `/gatherings/new`, `/gatherings/[id]`

## J6 — 관리자 회원 승인·관리

- **시작 조건**: 관리자 계정 + 승인 대기 신규 가입 1건 · 데스크톱
- **단계**: 1. 승인 대기 확인 2. 승인/거절 3. 회원 목록 반영 4. 당사자 알림 확인
- **채점 화면**: `/admin/approvals`, `/admin/members`
