- [ ] **QS-4 cancelTransaction 가드 시맨틱 버그 (잠복 — undo 복원 전 수정 필수)** — P1 · 저니 J2 · 서버 액션 · 기준 2.신뢰(부분 실패 정합)
  가드가 `cfm_at(확정 벽시계) <= snap.last_calc_at`로 "이미 재계산 반영" 여부를 판정하지만, `last_calc_at`은 재계산 시각이 아니라 **마지막 은행거래 시각 커서**다(RPC 주석: "입금 커서"). 통상 흐름(확정 직후 재계산)에서 cfm_at(현재) > last_calc_at(과거 거래시각) → 가드가 통과되어, 스냅샷에 이미 흡수된 납부를 취소해도 잔액은 그대로 남는다(부풀림). 취소된 거래는 인박스로 되돌아와 재확정도 가능 → 원장 이중행.
  (스윕 #1 2026-07-02 · 근거: `cancel-transaction.ts:18`, `recalculate-balance.ts:196`, `20260628100300_recalc_member_balance_rpc.sql:20`)

- [ ] **QS-5 확정 루프 부분 실패 시 보상 없음 — "확정됐지만 원장 미기록" 상태 가능** — P1 · 저니 J2 · 서버 액션 · 기준 2.신뢰(부분 실패 정합)
