# 모임 등록 편의기능 — 정기런 템플릿 & 최근글 불러오기

- 날짜: 2026-07-02
- 브랜치: `feature/meeting-registration-enhancements`
- 대상 폼: `components/schedule/gathering-form-dialog.tsx` (실사용 유일 모임 등록 폼)

## 배경 / 목적

모임 등록 시 반복 입력 부담을 줄이는 두 가지 편의기능을 추가한다.

1. **정기런 템플릿** — 정기런은 매번 내용이 거의 같고 장소/일시만 바뀐다. 유형에서 "정기"를 고르면 제목·비고·시작시간을 기본값으로 자동 채운다.
2. **최근글 불러오기** — 같은 내용의 모임을 반복 등록하는 사용자를 위해, 본인이 이전에 만든 모임을 골라 폼에 채운다.

## 범위 (Scope)

- **신규 테이블/마이그레이션 없음.** 템플릿은 코드 상수, 최근글은 기존 `gthr_mst` 조회.
- 관리자 편집 UI는 이번 범위에서 제외 (문구 수정은 상수 파일 PR로). 추후 필요 시 전용 `_cfg` 테이블 추가로 확장.
- 죽은 코드 `app/(info)/gatherings/new/gathering-form.tsx`는 손대지 않는다 (현재 `/gatherings/new`는 홈으로 redirect).

## 아키텍처

수정/추가 파일:

| 파일 | 상태 | 역할 |
|------|------|------|

## 기능 1 — 정기런 템플릿 자동 채움

### 트리거
유형(`gthr_type_enm`) 드롭다운에서 **정기(`regular`)** 를 선택하는 순간(`onValueChange`) 실행.

### 채움 규칙
- **제목(`gthr_nm`)**: 비어있을 때만 → `🔥 정기 기강런`
- **비고(`desc_txt`)**: 비어있을 때만 → 아래 상수 문구
- **시작시간(`stt_at`)**: 사용자가 시간을 직접 건드리지 않았을 때만 → 날짜는 유지하고 시각만 **19:30**
  - "건드렸는지" 판별: react-hook-form `formState.dirtyFields.stt_at` (falsy면 폼 기본값 그대로 → 교체)
  - **알려진 예외**: `useFormPersist`가 sessionStorage에서 복원할 때 `form.reset()`을 호출하면 dirty 상태가 초기화된다. 따라서 "시간 변경 → 앱 백그라운드 → 복원 → 정기 선택" 경로에서는 변경한 시간이 19:30으로 덮일 수 있다. 흔한 경로가 아니라 `dirtyFields` 방식으로 진행하되, 문제되면 datetime 피커 onChange에 별도 touched 플래그를 두어 보강한다.
- 이미 사용자가 입력한 값은 덮어쓰지 않는다.
- 다른 유형으로 다시 바꿔도 자동으로 채운 값을 강제로 지우지 않는다 (사용자가 지우는 것은 자유).

### 비고 템플릿 문구 (상수)

일시·장소는 폼 필드에 별도로 있으므로 문구에서 제외. 맨 앞에 강조 마크 하나(`✳`).

```
✳ 정기러닝 안내 🏃

즐거운 모임을 위해 정기러닝을 진행합니다. 🎉
번개와 공지로만 운영하던 모임에 정기적인 만남의 날을 마련해,
기강인들끼리 더 자주 교류하고 신규 인원도 부담 없이
참여할 수 있는 자리를 만들어갑니다. 😊

🏞 러닝 코스
· 기본 코스 7K (당일 상황에 따라 조정)
· 새싹 코스 4K (당일 상황에 따라 조정)

⚠️ 유의사항
· 코스는 날씨·현장 상황에 따라 변경될 수 있습니다.
```

### 상수 정의 (`lib/gathering/templates.ts`)


## 기능 2 — 최근글 불러오기

### 서버 액션 `getRecentGatherings()`
- `app/actions/gathering/get-recent-gatherings.ts`, `withMember` 사용.
- `gthr_mst`에서 `crt_by = 현재 멤버`, `del_yn = false`, `stt_at`(또는 `crt_at`) 최신순으로 넉넉히(예: 30개) 조회.
- **중복 제거**: 아래 시그니처가 모두 같은 모임은 가장 최근 것 하나로 접는다.
  - 시그니처 = `gthr_nm` + `gthr_type_enm` + `sprt_cd` + `loc_txt` + `max_prt_cnt` + `desc_txt` (일시 제외).
  - null/빈 문자열은 동일하게 정규화해 비교 (예: `loc_txt` null == "").
- 중복 제거 후 **최대 8개** 반환.
- 반환 필드: `gthr_id, gthr_nm, gthr_type_enm, sprt_cd, loc_txt, desc_txt, max_prt_cnt, stt_at`
  - `stt_at`은 목록 표시(날짜 라벨)용이며 폼에는 주입하지 않는다.

### UI (폼 상단 버튼 → 목록 팝업)
- `GatheringFormDialog`가 `mode === "create"`일 때만 폼 상단에 `최근 모임 불러오기` 버튼 노출.
- 클릭 시 별도 Dialog(또는 중첩 목록)에서 최근 모임을 **제목 + 유형·날짜·장소**로 나열.
  - 버튼 클릭 시점에 `getRecentGatherings()` 호출 (지연 로딩). 로딩/빈 목록 상태 처리.
- 항목 선택 시 폼에 주입(덮어쓰기 — 사용자가 명시적으로 불러온 행위이므로):
  - **제목, 유형, 종목, 장소, 최대인원, 비고**
  - **시작/종료 일시는 주입하지 않음** (새 모임 기준).
- 주입 후 팝업 닫기.

## 데이터 흐름

```
[유형 = 정기 선택]
  → onValueChange("regular")
  → 제목 empty? → REGULAR_GATHERING_TEMPLATE.title
  → 비고 empty? → REGULAR_GATHERING_TEMPLATE.desc
  → stt_at !dirty? → 선택 날짜 + 19:30

[최근 모임 불러오기 버튼]
  → getRecentGatherings() (서버 액션, crt_by=me)
  → 팝업 목록(제목+날짜)
  → 항목 선택 → form.setValue(제목/유형/종목/장소/최대인원/비고)
```

## 성능 원칙 (사용자 부하 최소화)
- **지연 로딩**: `getRecentGatherings()`는 "최근 모임 불러오기" 버튼을 누를 때만 호출. 폼 진입 시엔 조회하지 않는다.
- **1회 조회 캐시**: 한 번 불러온 목록은 컴포넌트 상태에 보관 → 팝업을 다시 열어도 재조회하지 않는다 (다이얼로그 세션 동안 유효).
- **인덱스 활용**: `crt_by` 필터는 기존 `ix_gthr_mst_crt_by` 부분 인덱스(`WHERE del_yn=false`)로 탐색.
- **작은 payload**: 필요한 컬럼만 select, 최근 30개만 조회, 서버에서 중복 제거해 최대 8건만 클라이언트로 전달.

## 에러 처리
- `getRecentGatherings()` 실패 시 팝업에 "불러오기 실패" 표시, 폼은 그대로.
- 최근 모임 0개면 빈 상태 메시지.
- 템플릿 자동 채움은 순수 클라이언트 동작이라 실패 지점 없음.

## 테스트 관점
- 정기 선택: (a) 빈 폼 → 제목·비고·19:30 채움 (b) 제목만 입력된 상태 → 제목 유지, 비고만 채움 (c) 시간 직접 변경 후 정기 선택 → 시간 유지.
- 최근글: 본인이 만든 모임만 나오는지(다른 멤버 것 제외), 삭제된 모임 제외, 선택 시 지정 필드만 채워지고 일시는 안 채워지는지.
