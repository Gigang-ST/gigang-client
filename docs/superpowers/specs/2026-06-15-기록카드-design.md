# 기록 카드 (Record Card) 설계

> 작성일: 2026-06-15 · 상태: 승인 대기

## 목적

러닝크루 "기강" 멤버 개인의 기록을 **수집형 트레이딩 카드** 형태로 보여주고 공유하는 재미 요소.
랭킹에서 이름을 누르면 그 사람의 카드가 팝업으로 뜨고, 본인은 자기 카드를 JPG로 저장할 수 있다.
PR(개인기록) 자랑·인스타 공유를 통한 크루 활동 동기 부여가 목적이며, 기능적 필요(URL 프로필 등)는 없다.

## 카드 구성

C 방향(트레이딩 카드) · **사진 풀블리드** 레이아웃.

```
┌─────────────────────────┐
│                         │  ← 프로필 사진이 카드 상단을 꽉 채움
│      [프로필 사진]       │     (하단 그라디언트 오버레이)
│                         │
│  김기강                  │  ← 이름 (사진 위 얹힘)
│  ✦ 새벽의 페이서 ✦       │  ← 대표 칭호 + 이펙트
├─────────────────────────┤
│  SELECTED RECORDS       │
│  ● 로드 · FULL  3:24:18 │
│  ● 로드 · 10K     42:30 │  ← 선택한 (종목 × 거리)별 최고기록
│  ● 트레일 · 50K 6:12:40 │     선택한 만큼 모두 표시(상한 없음)
│  ● 철인 · 올림픽 2:38:11│
└─────────────────────────┘
```

- 카드 테마: 앱 라이트/다크를 그대로 따름(화이트 카드 = 취향 차이, 둘 다 정상).
- **등급(rarity) 배지: 없음** (논의 후 폐기).

## 기존 컬렉션 시스템 재사용 (핵심 원칙)

새 디자인 토큰을 거의 만들지 않고 이미 있는 자산을 그대로 입힌다.

| 카드 요소 | 출처 | 비고 |
|-----------|------|------|
| 카드 프레임 | `team_mem_rel.selected_frame_cd` → `globals.css`의 `.card-frame-*` | 멤버가 장착한 프레임 그대로 |
| 칭호 이펙트 | `team_mem_rel.selected_badge_effect` → `.title-effect-*` | 멤버가 장착한 이펙트 그대로 |
| 대표 칭호명 | `mem_ttl_rel.is_prmy_yn = true` → `ttl_mst.ttl_nm` | 없으면 칭호 줄 생략 |
| 이름 / 사진 | `mem_mst.mem_nm` / `mem_mst.avatar_url` | |

"내가 컬렉션에서 꾸민 그대로가 곧 카드"가 되어 수집·꾸미기 동기와 직결된다.

## 기록 선택 (골라 담기)

- 본인 편집 화면에서 보유한 **(종목 × 거리)** 조합 목록을 ✓ 토글한다.
- 종목: `comp_mst.comp_sprt_cd` (road_run / trail_run / triathlon / cycling)
- 거리/이벤트: `comp_evt_cfg.comp_evt_type` (FULL/HALF/10K, 50K, 올림픽 …)
- 각 셀 값 = 해당 (종목, 거리)의 **최고 기록** = `min(rec_race_hist.rec_time_sec)`.
- **표시 개수 상한 없음** — 고른 만큼 모두, 넘치면 카드 세로로 늘어남/스크롤.
- 마라톤 기록만 있는 사람은 로드 항목만 노출. 철인·트레일 기록이 생기면 목록에 자동 추가되어 골라 담을 수 있다.

### 저장
- `team_mem_rel`에 신규 컬럼 `card_featured jsonb` (예: `[{"sport":"road_run","evt":"FULL"}, ...]`).
  - `selected_frame_cd` / `selected_badge_effect` 와 동일한 패턴.
- `null`(미설정) = 기본값으로 보유 기록 전부 또는 종목별 대표 기록 자동 표시.
- 저장은 서버 액션(`app/actions/`)으로 본인 row만 갱신.

## 진입점 / 권한

| 위치 | 동작 | 권한 |
|------|------|------|
| 랭킹(`/records`) 이름 클릭 | 카드 팝업(`Dialog`) — **보기 전용** | 누구나(공개) |
| 본인 프로필(`/profile`) | 내 카드 + 기록선택 편집 + **JPG 저장** | 본인 |
| 남의 카드 | 그 사람이 고른 기록 그대로 표시, 다운로드 버튼 없음 | 누구나(공개) |

- 카드는 **공개 안전 데이터만** 노출: 이름·아바타·대표칭호·프레임/이펙트·선택 기록.
  전화번호·이메일·생일·회비 등은 절대 포함하지 않는다.
- 랭킹 자체가 이미 공개(`get_public_team_race_rankings`)이므로 기록 공개는 기존 정책과 일관.

## 데이터 접근

신규 공개 RPC: `get_public_member_card(p_mem_id uuid, p_team_id uuid)`
- 반환: `mem_nm`, `avatar_url`, 대표 칭호명, `selected_frame_cd`, `selected_badge_effect`,
  그리고 해당 멤버의 `card_featured`에 해당하는 (종목, 거리)별 최고기록 목록.
- 서버에서 featured만 resolve 해서 내려보내 payload 최소화.
- RLS/보안: 민감 컬럼 미포함 확인, `del_yn=false` / 활성 멤버만, team scope 강제.

## 컴포넌트 구조

| 파일 | 역할 | 종류 |
|------|------|------|
| `components/records/record-card.tsx` | 카드 본체(프레젠테이셔널) — props만 받아 렌더 | 공유 |
| `components/records/record-card-dialog.tsx` | `mem_id`로 데이터 패칭 후 카드 팝업 | client |
| `components/profile/my-record-card.tsx` | 내 카드 + 편집 진입 + 다운로드 버튼 | client |
| `components/profile/card-record-picker.tsx` | (종목×거리) ✓ 선택 UI(Dialog/Sheet) | client |
| `lib/queries/member-card.ts` | `getPublicMemberCard()` 등 조회 | server |
| `app/actions/...` | `card_featured` 저장 서버 액션 | server |

- `record-card.tsx`는 순수 표현 컴포넌트로 분리해 랭킹 팝업·본인 카드·다운로드 캡처가 동일 DOM을 공유한다.
- 랭킹 이름 클릭은 `records-client.tsx`(client)에서 dialog state를 열고 `mem_id`를 넘긴다.

## 다운로드 (JPG)

- 라이브러리: **`html-to-image`** (`toJpeg`).
  - SVG `foreignObject` 기반이라 oklch·CSS 변수·`.card-frame-*` 효과를 그대로 캡처(html2canvas는 oklch에서 깨짐).
  - 애니메이션은 캡처 시점의 한 프레임으로 고정 — 저장 용도엔 문제없음.
- 본인 카드에서만 노출. 파일명 예: `기강카드_김기강_2026.jpg`.
- 아바타 CORS: Supabase 퍼블릭 버킷이면 그대로, 아니면 fetch→dataURL 변환 후 캡처.
- 캡처 대상은 현재 화면 테마(라이트/다크) 그대로.

## 범위 밖 (YAGNI)

- 외부 공유 URL / OG 이미지 서버 렌더 — 불필요(자랑·저장이 목적).
- 등급/레벨/포인트 시스템 — 폐기.
- 카드 기록 개수 상한 — 두지 않음.

## 미해결/구현 시 확인

- `card_featured` 기본값(미설정) 정확한 규칙: "전부" vs "종목별 대표" — 구현 단계에서 단순한 쪽으로 확정.
- 대표 칭호 없는 멤버의 칭호 줄 처리(생략).
- 기록 0개 멤버 카드의 빈 상태 문구.
