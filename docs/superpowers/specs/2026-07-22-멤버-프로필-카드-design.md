# 멤버 프로필 카드 (쇼케이스) 설계

- 작성일: 2026-07-22
- 선행 문서: [기강이야기 전광판 설계서](2026-07-15-기강이야기-전광판-design.md) §6 — "멤버 프로필 카드는 별도 설계"로 미뤄둔 바로 그 문서
- 관련 문서: [칭호 시스템 설계](../../design/2026-03-23-칭호시스템.md)

## 1. 목적

랭킹·모임 참석자·(추후) 기강 전광판에서 **사람을 탭하면 그 멤버의 "게임 유저정보창"이 뜬다.**
칭호·기록·활동으로 쌓아온 그 사람의 크루 정체성을 한 장으로 보여주는 카드이며,
기강이야기 전광판 스포트라이트의 원자 단위(선행 의존성)이기도 하다.

두 가지 버전을 만든다:

| 버전 | 쓰임 | 형태 |
|---|---|---|
| **컴팩트 카드** | 전광판 스포트라이트, 피드/리스트 인라인 | 아바타 + 이름 + 대표칭호 + 한마디 |
| **상세 카드** | 랭킹·모임·전광판에서 사람 탭 시 | 중앙 다이얼로그 (유저정보창) |

## 2. 결정 사항 (브레인스토밍 확정)

1. **컨셉**: 게임 유저정보창 — 정보 중심 스탯 패널형. (트레이딩카드+JPG 저장 컨셉은 폐기)
2. **상세 진입**: 중앙 다이얼로그.
3. **상세 콘텐츠**: 기본(사진·이름·대표칭호·한마디·종목별 베스트기록) + 보유 칭호 전체 + 활동 스탯 + 활동지수 + 가입일·함께한 기간. 전부 포함.
4. **record-card 잔재 정리**: ✅ 2026-07-22 완료 —
   - dev의 `apply_team_mem_rel_change` / `apply_team_mem_rel_delete`를 prd 본문으로 복원(`card_featured` 운반 제거)
   - dev의 `get_public_member_card(uuid,uuid)` RPC drop
   - dev의 `team_mem_rel.card_featured` 컬럼 drop (마이그레이션 히스토리 미등록 드리프트였음 — 객체 drop만으로 정리 종료)
   - `origin/feature/record-card` 브랜치 삭제
   - prd에는 애초에 아무것도 없었음 (확인 완료)

## 3. 카드 구성 (anatomy)

### 3.1 상세 카드 (중앙 다이얼로그) — "라인업 콜" 구도

야간 스타디움 선수 소개판. 카드 상단 ~44%는 **테마와 무관하게 항상 어두운 스크린 존**(전광판 유리),
그 아래는 기존 `--card` 배경의 정보 존. (컨셉 근거는 §6)

```text
╭═[장착 card-frame-* = 경기장 외곽 조명]═╮
│▓▓▓ 스크린 존 — --board, 라이트/다크 공통 다크 ▓▓▓
│ 📣 춘천마라톤 D-3 출격  (앰버 스트립)  │  ← 출격 예고 (D-7 이내 등록 대회 시)
│ NO.27                        ● LIVE   │  ← 등번호(합류 순번) · 앰버 Micro
│        ╱ 스포트라이트 콘 ╲             │
│         (아바타 XL 88px)               │
│          김  기  강                    │  ← 명조 800 24px — 선수 콜
│       ✦ 새벽의 페이서 ✦                │  ← title-effect-* 원본 발광
│      "올해는 서브4 간다"                │  ← 한마디 — 명조 인용 15px
│  26.03.12 합류 ······· 132일째         │  ← 도트리더 Micro
├─ RECORDS ──────(이하 테마 --card)─────┤
│ ● 로드 FULL ········ 3:24:18 [NEW]    │  ← sport 도트 · mono · NEW 블링크
│ ● 로드 10K ·········· 42:30           │
│ ● 트레일 UTMB ·········· 512          │
├─ ACTIVITY ───────────────────────────┤
│ [참석 12] [출전 3] [함성 872㏈ ▮▮▮▯]   │  ← StatCard 3열, 3번째 = 함성 미터
├─ TITLES (5) ─────────────────────────┤
│ [고인물] [산악구보] [맛객] …           │  ← 보유분만, 희소도 내림차순, 툴팁
╰───────────────────────────────────────╯
```

- 기록 0개 → "아직 등록된 기록이 없습니다." / 칭호 0개 → TITLES 존 생략 / 한마디 미설정 → 줄 생략(placeholder 없음).
- 본인이 본인을 탭해도 동일 카드(편집 기능 없음 — 편집은 프로필 탭·컬렉션에서).

### 3.2 컴팩트 카드 (인라인)

```text
╭─[프레임]───────────────────────╮
│ (아바타 lg)  김기강             │
│  ✦새벽의 페이서✦               │
│  "올해는 서브4 간다"            │
╰───────────────────── 탭→상세 ──╯
```

- **기록·숫자는 넣지 않는다** — 전광판에서는 이유 배너가 헤드라인·숫자를 얹으므로 중복 방지. 정보는 상세의 몫.
- 탭하면 상세 다이얼로그.

## 4. 데이터 / 백엔드

### 4.1 신설 마이그레이션 (정식 — dev·prd 공통 배포)

**① `team_mem_rel.intro_txt text`** — "한마디". `check (char_length(intro_txt) <= 60)`.
팀 스코프 꾸미기 자산(`selected_frame_cd` 패턴)과 동일한 위치.

> ⚠️ **상태이력 함수 동반 갱신 필수**: `apply_team_mem_rel_change` / `apply_team_mem_rel_delete`는
> 명시적 컬럼 목록으로 이력 스냅샷을 INSERT 한다. `team_mem_rel`에 컬럼을 추가하는 마이그레이션은
> **반드시 같은 마이그레이션에서 두 함수의 컬럼 목록에 `intro_txt`를 추가**해야 한다
> (record-card 브랜치가 dev에서만 이걸 하고 미머지되어 드리프트가 났던 전례).

**② `get_public_member_card(p_mem_id uuid, p_team_id uuid) returns jsonb` 재정의(v2)**
같은 이름을 정식 마이그레이션으로 부활 — 전광판 설계서 §6의 참조가 그대로 유효하다.
`STABLE SECURITY DEFINER, search_path=public`. 랭킹 공개 정책과 일관되게 anon 호출 허용.

```jsonc
{
  "mem_nm": "김기강",
  "avatar_url": "...",                    // mem_mst
  "badge_effect": "gold",                 // team_mem_rel.selected_badge_effect (없으면 "none")
  "frame_cd": "frame-aurora",             // team_mem_rel.selected_frame_cd (없으면 "frame-none")
  "intro_txt": "올해는 서브4 간다",         // 신설 — 한마디
  "join_dt": "2026-03-12",                // team_mem_rel.join_dt — "합류 · N일째" 계산용
  "back_no": 27,                          // 등번호 — rank() over (order by join_dt, mem_id). 탈퇴자 포함 전체 합류 순번으로 고정(탈퇴 시 번호 당겨짐 방지)
  "upcoming_race": {                      // D-7 이내 시작하는 등록 대회 최근접 1건, 없으면 null — 출격 예고 스트립용 (comp_reg_rel × comp_mst 조인)
    "comp_nm": "춘천마라톤", "start_dt": "2026-07-25"
  },
  "utmb_index": 512,                      // mem_utmb_prf (nullable)
  "primary_title": { "ttl_nm": "...", "ttl_desc": "...", "desc_visibility": "others" },
  "titles": [                             // 보유분 전체, rarity_level 내림차순
    { "ttl_nm": "...", "ttl_desc": "...", "desc_visibility": "...", "rarity_level": 5, "ttl_ctgr_cd": "running" }
  ],
  "best_records": [                       // (종목×거리)별 최고기록 — v1 로직 유지, card_featured 없음
    { "sport": "road_run", "evt": "FULL", "rec_time_sec": 12258, "race_nm": "...", "race_dt": "..." }
  ],
  "stats": {
    "gthr_attd_cnt": 12,                  // 모임 유효 참석 누적 (취소 참석 제외)
    "comp_reg_cnt": 3,                    // 종료된 대회 출전 누적
    "activity_score": 872                 // pt_txn_hist 합산 — "활동지수"로만 노출
  }
}
```

- **반환 `null`** = 카드 없음: 해당 팀 `team_mem_rel` 없음, 탈퇴(left)·비활성(inactive), `del_yn=true`.
  → 클라이언트는 "함께 달렸던 멤버예요" 폴백(이름만). left/inactive 사유는 구분 노출하지 않는다.
- 모든 서브쿼리 `vers=0, del_yn=false` 기본.

### 4.2 프라이버시 (공개 허용목록)

- 노출: 이름·아바타·칭호(보유분)·장착 프레임/이펙트·종목별 베스트기록·UTMB 인덱스·참석/출전 횟수·활동지수·합류일·한마디.
- **비노출 고정**: 성별·생일·연락처·이메일·거주지(가까운 역)·회비/계좌·온보딩 응답. 랭킹이 이미 공개이므로 기록 공개는 기존 정책과 일관.
- **기강 포인트 히든 원칙 준수**: `pt_txn_hist` 합산을 "활동지수" 숫자 하나로만 노출. "포인트" 표기·원장 구조·적립 규칙은 UI에 노출하지 않는다 (전광판 설계서 §11-1 확정과 동일).

## 5. 프론트엔드 구조

| 파일 | 역할 | 종류 |
|---|---|---|
| `lib/queries/member-card.ts` | `getPublicMemberCard()` RPC 호출 + `MemberCardData` 타입/파서 | 공유 |
| `components/members/member-card.tsx` | 컴팩트 카드 (순수 표현 — props만 렌더) | 공유 |

| `components/members/member-card-dialog.tsx` | `memId`·`teamId`로 오픈 시 fetch + Dialog + 스켈레톤/폴백 | client |

- 데이터 로딩: **원샷** — 다이얼로그 오픈 시 RPC 1회. 오픈 전 prefetch 없음.
- 컴팩트 카드는 데이터를 props로만 받는다 — 전광판 피드 RPC가 payload를 내려주면 fetch 없이 그린다.

### 한마디 편집

- `/profile/edit` 폼에 "한마디" 입력 추가 — zod 스키마(`lib/validations/`, 60자) + RHF, 기존 서버 wrapper + client form 패턴.

## 6. 디자인 방향 — "라인업 콜" (야간 스타디움 선수 소개판)

[DESIGN.md](../../../DESIGN.md) 준수. 개성은 새 장식이 아니라 **이미 가진 게임 자산**에서 나온다.
컨셉: **탭하는 순간 장내 아나운서가 이 선수를 부르고 전광판이 점등된다** — 전광판 설계서(스타디움 점보트론)와
같은 세계관의 "선수 소개판". (2026-07-22 디자인 판정단 워크플로 채택안)

### 6.1 시그니처: 항상 야간인 스크린 존

- 카드 상단 ~44%만은 **라이트 모드에서도 어두운 전광판 유리**로 남는다. 기능적 근거: `globals.css`의
  장착 프레임 22종(`card-frame-*`)·칭호 이펙트 48종(`title-effect-*`)은 glow 기반이라 흰 배경에서 발광이
  죽는다 — 항상-다크 스크린 존은 컬렉션 보상이 **두 테마 모두에서 설계 의도대로 발광하는 유일한 무대**다.
  장식을 새로 만드는 게 아니라 기존 자산이 가장 잘 사는 조도를 만들어 주는 시그니처.
- **신설 토큰 5종** (globals.css + DESIGN.md 등재 필요, 스크린 존 전용):
  `--board`(전광판 유리 #15171e / 다크 #101218) · `--board-foreground` · `--board-muted` ·
  `--board-line`(도트리더·구분선) · `--board-amber`(#f5b942 — LIVE 도트·NEW 태그·함성 미터·등번호
  점등에만 제한 사용. **스크린 존 밖 침투 금지** — "다크+네온" 템플릿이 아니라 전광판 앰버 LED라는 사물 근거).
- 스포트라이트 콘은 토큰이 아닌 알파 화이트 radial-gradient — 아바타를 비추는 조명.
- 스크린 존 밖(하단 존)은 기존 `--card`/`--border`/`--muted-foreground` 그대로. 종목 도트는 `sport-*` 토큰.

### 6.2 타이포 (기존 2종 폰트 내)

- **이름 = Nanum Myeongjo 800 24px** — 전광판 명조 헤드라인(전광판 스펙 §7)과 같은 혈통의 "선수 콜".
- 대표칭호 = Pretendard 600 13px + 장착 `title-effect-*` 원본 그대로 (다크 스크린 위라 글로우 최적).
- **한마디 = 명조 인용체 15px** (기존 확정 유지). 메타(합류·N일째)는 Micro + `--board-line` 도트리더(····) 연결.
- 기록·미터 숫자 = `font-mono tabular-nums`(기존 컨벤션), 등번호 NO.n = mono 700 12px `--board-amber`.
- 존 구분 `SectionLabel`, 스탯 `StatCard` 3열 재사용 (기존 확정 유지). 색·타이포 매직넘버 금지.

### 6.3 모션: 점등 시퀀스 (총 ~1.1s, 오픈 시 1회)

Dialog 기본 zoom-in → 스크린 존 형광등 플리커(280ms, 기존 `frame-lightning` 키프레임 문법 재사용) →
스포트라이트 콘 fade-in(200ms) → 이름·칭호·한마디 60ms 스태거 상승 → 함성 미터 카운트업(900ms —
기존 "활동지수 1회 카운트업" 확정을 이 연출로 수행). **아바타 탭 = 시퀀스 리플레이.**

- `prefers-reduced-motion`: 플리커·스태거·카운트업 전부 제거, 최종 상태 즉시 표시.
- 상시 루프 애니메이션은 장착 프레임/이펙트에만 양보 — 카드 자체는 정적. 과함 금지.

### 6.4 재미요소

| 요소 | 내용 | 데이터 | 단계 |
|---|---|---|---|
| **함성 데시벨 미터** | `activity_score`를 "함성 ㏈"로 재포장 — 세로 바 점등 + 카운트업. 등급 5종: 워밍업 → 트랙인(10+) → 함성(30+) → 떼창(60+) → 지축(100+). **구간은 실측 분포 기준**(아래 주) | `stats.activity_score` (기존) | 1차 |

## 10. 구현 시 확인 사항 — ✅ 1차 구현에서 확정된 답

> 아래 1~4는 2026-07-22 구현하며 실제 코드·DB로 확인한 결과다.

1. **유효 참석 판정** — `gthr_attd_rel`에는 취소 플래그가 없다. 취소는 행을 지우고 `gthr_attd_hist`에 이벤트로 남긴다(`evt_cd`). 따라서 **살아있는 행 = 유효 참석**이며, 여기에 `gthr_mst.stt_at < now()`(이미 시작한 모임)만 더했다.

### 구현 중 추가로 드러난 것

5. **`TitleBadge`가 전파를 막지 않았다** — 배지 탭이 부모로 새어 툴팁과 부모 액션이 동시에 열렸다. `handleClick`에 `stopPropagation()`을 추가(공용 수정). 트레일 랭킹은 이름이 이미 UTMB 외부 링크라, 링크에도 전파 차단을 걸고 행의 나머지 영역이 카드를 연다.
6. **명조가 실제로는 로드되지 않고 있었다** — DESIGN.md는 Nanum Myeongjo를 제목 폰트로 명시하지만 `app/layout.tsx`는 Inter만 로드했다. "한마디 = 명조 인용체" 시그니처가 시스템 serif로 조용히 폴백되므로, `next/font/google`로 정식 로드하고 `--font-serif`를 연결했다(`preload: false` — 한글 서브셋이 커서 쓰는 화면에서만 내려받게).
7. **Tailwind v4 스캐너와 이모지** — 소스에 astral-plane 이모지(📣 U+1F4E3)가 있으면 클래스 후보 언이스케이프 과정에서 `RangeError: Invalid code point`로 **빌드가 CSS 에러처럼 실패**한다(에러는 globals.css:1을 가리켜 오해하기 쉽다). 스크린 존 스트립의 이모지는 `lucide-react`의 `Megaphone` 아이콘으로 교체했다.
8. **`card_featured` 드리프트 해소** — 리포의 `20260715121000/122000` 마이그레이션 파일은 실 DB에 없는 `card_featured`를 아직 참조하는 stale 버전이다. 이력 함수를 재정의할 때 그 파일을 베이스로 삼으면 깨지므로 **실 DB 본문(`pg_get_functiondef`) 기준**으로 작성했다. 타입 재생성으로 `database.types.ts`의 잔재도 정리됐다.
