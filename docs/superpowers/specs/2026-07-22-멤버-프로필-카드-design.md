# 멤버 프로필 카드 (쇼케이스) 설계

- 작성일: 2026-07-22
- 선행 문서: [기강이야기 전광판 설계서](2026-07-15-기강이야기-전광판-design.md) §6 — "멤버 프로필 카드는 별도 설계"로 미뤄둔 바로 그 문서
- 관련 문서: [칭호 시스템 설계](../../design/2026-03-23-칭호시스템.md)

---

## 1. 목적

랭킹·모임 참석자·(추후) 기강 전광판에서 **사람을 탭하면 그 멤버의 "게임 유저정보창"이 뜬다.**
칭호·기록·활동으로 쌓아온 그 사람의 크루 정체성을 한 장으로 보여주는 카드이며,
기강이야기 전광판 스포트라이트의 원자 단위(선행 의존성)이기도 하다.

두 가지 버전을 만든다:

| 버전 | 쓰임 | 형태 |
|---|---|---|
| **컴팩트 카드** | 전광판 스포트라이트, 피드/리스트 인라인 | 아바타 + 이름 + 대표칭호 + 한마디 |
| **상세 카드** | 랭킹·모임·전광판에서 사람 탭 시 | 중앙 다이얼로그 (유저정보창) |

## 2. 결정 사항 (브레인스토밍 확정)

1. **컨셉**: 게임 유저정보창 — 정보 중심 스탯 패널형. (트레이딩카드+JPG 저장 컨셉은 폐기)
2. **상세 진입**: 중앙 다이얼로그.
3. **상세 콘텐츠**: 기본(사진·이름·대표칭호·한마디·종목별 베스트기록) + 보유 칭호 전체 + 활동 스탯 + 활동지수 + 가입일·함께한 기간. 전부 포함.
4. **record-card 잔재 정리**: ✅ 2026-07-22 완료 —
   - dev의 `apply_team_mem_rel_change` / `apply_team_mem_rel_delete`를 prd 본문으로 복원(`card_featured` 운반 제거)
   - dev의 `get_public_member_card(uuid,uuid)` RPC drop
   - dev의 `team_mem_rel.card_featured` 컬럼 drop (마이그레이션 히스토리 미등록 드리프트였음 — 객체 drop만으로 정리 종료)
   - `origin/feature/record-card` 브랜치 삭제
   - prd에는 애초에 아무것도 없었음 (확인 완료)

## 3. 카드 구성 (anatomy)

### 3.1 상세 카드 (중앙 다이얼로그)

```
╭─[장착 프레임 이펙트가 카드 테두리]─╮
│         (아바타 XL, 중앙)         │  ← 게임 상태창 구도: 상단 중앙 정렬
│    김기강   ✦새벽의 페이서✦        │  ← 이름 + 대표칭호(장착 이펙트)
│    "올해는 서브4 간다"             │  ← 한마디 — 나눔명조 인용체
│    26.03.12 합류 · 132일째 달리는 중│
├─ RECORDS ────────────────────────┤
│ ● 로드 FULL       3:24:18        │  ← 종목 도트 = sport-* 토큰
│ ● 로드 10K          42:30        │     숫자 = font-mono tabular
│ ● 트레일 UTMB         512        │
├─ ACTIVITY ───────────────────────┤
│  [참석 12]  [출전 3]  [지수 872]   │  ← 3열 미니 스탯 (StatCard)
├─ TITLES (5) ─────────────────────┤
│ [고인물] [산악구보] [맛객] …       │  ← 보유분만, 희소도 내림차순,
╰──────────────────────────────────╯     TitleBadge 툴팁으로 획득조건
```

- 기록 0개 → "아직 등록된 기록이 없습니다." / 칭호 0개 → TITLES 존 생략 / 한마디 미설정 → 줄 생략(placeholder 없음).
- 본인이 본인을 탭해도 동일 카드(편집 기능 없음 — 편집은 프로필 탭·컬렉션에서).

### 3.2 컴팩트 카드 (인라인)

```
╭─[프레임]───────────────────────╮
│ (아바타 lg)  김기강             │
│  ✦새벽의 페이서✦               │
│  "올해는 서브4 간다"            │
╰───────────────────── 탭→상세 ──╯
```

- **기록·숫자는 넣지 않는다** — 전광판에서는 이유 배너가 헤드라인·숫자를 얹으므로 중복 방지. 정보는 상세의 몫.
- 탭하면 상세 다이얼로그.

## 4. 데이터 / 백엔드

### 4.1 신설 마이그레이션 (정식 — dev·prd 공통 배포)

**① `team_mem_rel.intro_txt text`** — "한마디". `check (char_length(intro_txt) <= 60)`.
팀 스코프 꾸미기 자산(`selected_frame_cd` 패턴)과 동일한 위치.

> ⚠️ **상태이력 함수 동반 갱신 필수**: `apply_team_mem_rel_change` / `apply_team_mem_rel_delete`는
> 명시적 컬럼 목록으로 이력 스냅샷을 INSERT 한다. `team_mem_rel`에 컬럼을 추가하는 마이그레이션은
> **반드시 같은 마이그레이션에서 두 함수의 컬럼 목록에 `intro_txt`를 추가**해야 한다
> (record-card 브랜치가 dev에서만 이걸 하고 미머지되어 드리프트가 났던 전례).

**② `get_public_member_card(p_mem_id uuid, p_team_id uuid) returns jsonb` 재정의(v2)**
같은 이름을 정식 마이그레이션으로 부활 — 전광판 설계서 §6의 참조가 그대로 유효하다.
`STABLE SECURITY DEFINER, search_path=public`. 랭킹 공개 정책과 일관되게 anon 호출 허용.

```jsonc
{
  "mem_nm": "김기강",
  "avatar_url": "...",                    // mem_mst
  "badge_effect": "gold",                 // team_mem_rel.selected_badge_effect (없으면 "none")
  "frame_cd": "frame-aurora",             // team_mem_rel.selected_frame_cd (없으면 "frame-none")
  "intro_txt": "올해는 서브4 간다",         // 신설 — 한마디
  "join_dt": "2026-03-12",                // team_mem_rel.join_dt — "합류 · N일째" 계산용
  "utmb_index": 512,                      // mem_utmb_prf (nullable)
  "primary_title": { "ttl_nm": "...", "ttl_desc": "...", "desc_visibility": "others" },
  "titles": [                             // 보유분 전체, rarity_level 내림차순
    { "ttl_nm": "...", "ttl_desc": "...", "desc_visibility": "...", "rarity_level": 5, "ttl_ctgr_cd": "running" }
  ],
  "best_records": [                       // (종목×거리)별 최고기록 — v1 로직 유지, card_featured 없음
    { "sport": "road_run", "evt": "FULL", "rec_time_sec": 12258, "race_nm": "...", "race_dt": "..." }
  ],
  "stats": {
    "gthr_attd_cnt": 12,                  // 모임 유효 참석 누적 (취소 참석 제외)
    "comp_reg_cnt": 3,                    // 종료된 대회 출전 누적
    "activity_score": 872                 // pt_txn_hist 합산 — "활동지수"로만 노출
  }
}
```

- **반환 `null`** = 카드 없음: 해당 팀 `team_mem_rel` 없음, 탈퇴(left)·비활성(inactive), `del_yn=true`.
  → 클라이언트는 "함께 달렸던 멤버예요" 폴백(이름만). left/inactive 사유는 구분 노출하지 않는다.
- 모든 서브쿼리 `vers=0, del_yn=false` 기본.

### 4.2 프라이버시 (공개 허용목록)

- 노출: 이름·아바타·칭호(보유분)·장착 프레임/이펙트·종목별 베스트기록·UTMB 인덱스·참석/출전 횟수·활동지수·합류일·한마디.
- **비노출 고정**: 성별·생일·연락처·이메일·거주지(가까운 역)·회비/계좌·온보딩 응답. 랭킹이 이미 공개이므로 기록 공개는 기존 정책과 일관.
- **기강 포인트 히든 원칙 준수**: `pt_txn_hist` 합산을 "활동지수" 숫자 하나로만 노출. "포인트" 표기·원장 구조·적립 규칙은 UI에 노출하지 않는다 (전광판 설계서 §11-1 확정과 동일).

## 5. 프론트엔드 구조

| 파일 | 역할 | 종류 |
|---|---|---|
| `lib/queries/member-card.ts` | `getPublicMemberCard()` RPC 호출 + `MemberCardData` 타입/파서 | 공유 |
| `components/members/member-card.tsx` | 컴팩트 카드 (순수 표현 — props만 렌더) | 공유 |
| `components/members/member-card-detail.tsx` | 상세 카드 본체 (순수 표현) | 공유 |
| `components/members/member-card-dialog.tsx` | `memId`·`teamId`로 오픈 시 fetch + Dialog + 스켈레톤/폴백 | client |

- 데이터 로딩: **원샷** — 다이얼로그 오픈 시 RPC 1회. 오픈 전 prefetch 없음.
- 컴팩트 카드는 데이터를 props로만 받는다 — 전광판 피드 RPC가 payload를 내려주면 fetch 없이 그린다.

### 진입점

| 위치 | 동작 | 단계 |
|---|---|---|
| 랭킹 `records-client.tsx` | 엔트리 행(이름 영역) 탭 → 다이얼로그. 칭호 배지 탭은 기존 툴팁 유지(이벤트 전파 차단) | 1차 |
| 모임 상세 `gathering-detail-dialog.tsx` | 참석자 아바타·이름 탭 → 다이얼로그 (다이얼로그 위 다이얼로그 — 기존 중첩 패턴 확인 후 필요시 교체 방식) | 1차 |
| 기강 전광판 `/story` | 스포트라이트 = 컴팩트 카드 + 이유 배너, 탭 → 상세 | 2차 (전광판 스펙) |
| 내 프로필 탭 `ProfileCard` | 한마디 표시 추가 (편집 유도) | 1차 |

### 한마디 편집

- `/profile/edit` 폼에 "한마디" 입력 추가 — zod 스키마(`lib/validations/`, 60자) + RHF, 기존 서버 wrapper + client form 패턴.
- 저장은 본인 `team_mem_rel` 갱신 — 장착 프레임/이펙트 저장 액션과 동일 경로 사용(구현 시 해당 액션이 직접 update인지 이력 함수 경유인지 확인 후 동일하게).

## 6. 디자인 방향

[DESIGN.md](../../../DESIGN.md) 준수. 개성은 새 장식이 아니라 **이미 가진 게임 자산**에서 나온다.

- **시그니처 = 장착 프레임·칭호 이펙트가 주인공**: 프레임 23종(`card-frame-*`) + 이펙트 47종(`title-effect-*`)이 카드의 개성 전부를 담당한다. 카드 자체는 절제된 무대 — 새 장식 토큰·배경 그래픽을 만들지 않는다.
- **한마디 = 나눔명조 인용체**: 전광판 헤드라인(명조 시그니처, 전광판 스펙 §7)과 같은 혈통으로 묶어 세계관 통일. 본문·메타는 Pretendard.
- **상단 중앙 정렬 구도**: 아바타를 카드 상단 중앙에 크게 — 앱의 좌측 정렬 리스트 문법과 의도적으로 다른, "상태창" 구도.
- 존 구분은 `SectionLabel`(RECORDS/ACTIVITY/TITLES), 기록 숫자는 `font-mono tabular-nums`, 종목 도트는 `sport-*` 토큰, 스탯은 `StatCard`(3열 그리드) 재사용.
- 모션: Dialog 기본 등장 + 활동지수 1회 카운트업 정도. `prefers-reduced-motion` 존중, 과함 금지.
- 색·타이포 매직넘버 금지 — 전부 토큰·타이포 컴포넌트.

## 7. 엣지케이스

| 상황 | 처리 |
|---|---|
| 탈퇴·비활성 멤버 탭 | RPC null → "함께 달렸던 멤버예요" 미니 폴백(이름만). 참가기록 이름 보존 원칙과 일관 |
| 기록 0 | RECORDS 존에 "아직 등록된 기록이 없습니다." |
| 칭호 0 | TITLES 존 생략 (대표칭호 줄도 생략) |
| 한마디 미설정 | 줄 생략 — placeholder 없음 |
| 비로그인 조회 | 허용 (랭킹 공개와 동일). 리액션 등 쓰기는 없음 — 카드는 읽기전용 |
| RPC 실패 | 다이얼로그 내 에러 문구 + 재시도 버튼 |

## 8. 단계

- **1차 (이번 구현)**: 마이그레이션(intro_txt + 이력함수 2개 갱신 + RPC v2) → 카드 컴포넌트 3종 + 쿼리 → 진입점(랭킹·모임) → 한마디 편집(/profile/edit) + 내 ProfileCard 표시 → 타입 재생성(`database.types.ts` — 정리로 사라진 `card_featured` 타입도 이때 함께 반영).
- **2차 (전광판 스펙에서)**: 스포트라이트 연동, 리액션.
- **나중**: 내 카드 이미지 공유(JPG) 재검토 — 폐기한 record-card 브랜치의 html-to-image 노하우는 KNOWLEDGE 참고용으로만.

## 9. 비범위 (YAGNI)

- JPG 저장·외부 공유 URL — 1차 제외.
- 기록 골라담기(`card_featured`식 선택) — 종목별 베스트 전부 자동 표시로 충분.
- 등급/레벨 시스템 신설 — 기존 rarity·칭호 체계로 충분.
- 전용 페이지 `/members/[id]` — 다이얼로그로 확정, 딥링크 필요해지면 재검토.

## 10. 구현 시 확인 사항

1. `gthr_attd_rel`의 취소 상태 판별 컬럼 — "유효 참석" 카운트 조건 확정.
2. `comp_reg_cnt`의 "종료된 대회" 판정 — `comp_mst.end_dt < KST 오늘` 기준(기존 KST 헬퍼 사용).
3. 장착 프레임/이펙트 저장 액션의 갱신 경로(직접 update vs 이력 함수) 확인 후 `intro_txt`도 동일 경로.
4. 모임 상세가 이미 Dialog인 경우의 중첩 UX — 중첩 허용 vs 교체 전환 중 기존 앱 패턴 따름.
