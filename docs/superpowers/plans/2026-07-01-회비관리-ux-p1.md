# 회비 관리 UX 혁신 P1 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 운영자가 매월 은행 거래내역 엑셀을 올린 뒤, 학습형 매칭으로 잔여 거래만 Triage Inbox에서 빠르게 분류·매칭하고 한 번에 확정+재계산하며, 잔액 원장에서 회원별 더/덜 냄을 한눈에 본다.

**Architecture:** 기존 회비 데이터 모델(`fee_txn_hist`→`fee_due_pay_hist`→`fee_mem_bal_snap`)은 유지. 신규 `fee_payer_alias`(입금자명→회원 학습)와 순수 함수 매칭 파이프라인(`lib/dues/normalize-payer-name.ts`, `lib/dues/match-payer.ts`)을 추가하고, `upload-xlsx.ts`가 이를 호출해 3버킷(자동완료/확인필요/제외)을 만든다. 확정과 재계산을 `confirm-and-recalc.ts` 하나로 합치고, 관리자 라우트를 3면(잔액 원장·Triage Inbox·프로젝트[P2])으로 재편한다.

**Tech Stack:** Next.js App Router, React 19, TypeScript 5, Supabase(Postgres + RPC), Tailwind v4, shadcn/ui, vitest, dayjs(`@/lib/dayjs`).

**참조 스펙:** `docs/superpowers/specs/2026-07-01-회비관리-ux-design.md`

## Global Constraints

- 멤버 인증: 서버 액션은 `withAdmin(async ({ member }) => {...})`(`@/lib/actions/auth`)로 감싸고, 페이지는 `verifyAdmin()`(`@/lib/queries/member`) 가드. teamId는 `getRequestTeamContext()`(`@/lib/queries/request-team`), DB는 `createAdminClient()`(`@/lib/supabase/admin`).
- 날짜/시간: `import { dayjs } from "@/lib/dayjs"`만 사용. `new Date()`·문자열 slice/replace 포맷 금지. KST(`Asia/Seoul`) 기준.
- 환경변수: `lib/env.ts`에서 import. `process.env` 직접 접근 금지.
- 패키지 매니저: `pnpm`. 경로 별칭 `@/*` → 프로젝트 루트.
- 마이그레이션 컨벤션: `public` 스키마, PK/FK/UK/IX/CK 접두 네이밍, `vers integer DEFAULT 0` + `del_yn boolean DEFAULT false` 소프트삭제, `crt_at`/`upd_at timestamptz DEFAULT now()` + `BEFORE UPDATE ... EXECUTE FUNCTION public.set_v2_upd_at()` 트리거, RLS `*_select_member`(authenticated, 팀원) + `*_mutate_admin`(owner/admin) 정책 + `GRANT ALL ... TO anon, authenticated, service_role`.
- UI 텍스트·주석 한국어. 타이포그래피·색·컴포넌트는 `DESIGN.md` 토큰/컴포넌트 사용(매직넘버 금지).
- `fee_txn_hist.fee_item_cd` CHECK 허용값(기존): `'due','expense','event_fee','goods','other'`. P1 "프로젝트" 분류는 `event_fee`/`goods`로 매핑(신규 enum 추가 없음). `due`=회비, 그 외=비회비.

```ts
// lib/dues/normalize-payer-name.ts

/**
 * 은행 입금자명을 매칭용으로 정규화한다(단일 출처).
 * 업로드 파서와 별칭 조회가 같은 정규화를 쓰도록 이 함수만 사용한다.
 *
 * 규칙: 공백 제거 → 선/후행 "회비" 토큰 제거 → 숫자·특수문자 제거
 *      → 영문 소문자화 → NFC 정규화.
 */
export function normalizePayerName(raw: string): string {
  let s = (raw ?? "").normalize("NFC").trim();
  s = s.replace(/\s+/g, "");
  // 선/후행에 붙은 "회비" 라벨 제거 (중간에 포함된 경우는 보존)
  s = s.replace(/^회비/, "").replace(/회비$/, "");
  // 한글/영문/숫자 외 제거 후, 숫자도 제거 (금액 등 노이즈)
  s = s.replace(/[^0-9A-Za-z가-힣]/g, "");
  s = s.replace(/[0-9]/g, "");
  return s.toLowerCase();
}
```

  ```ts
  export type MemberRef = { memId: string; name: string };
  export type AliasRef = { rawNameNorm: string; memId: string };
  export type MatchCandidate = { memId: string; name: string; score: number }; // score 0~1
  export type MatchStatus = "matched" | "ambiguous" | "unmatched";
  export type MatchResult = {
    status: MatchStatus;
    memId: string | null;              // matched일 때만 채움
    via: "alias" | "exact" | null;     // 자동매칭 출처
    candidates: MatchCandidate[];      // UI 후보(신뢰도 내림차순), 최대 5
  };
  export function matchPayer(rawName: string, members: MemberRef[], aliases: AliasRef[]): MatchResult;
  ```
  파이프라인: (1) 별칭 정확일치 단일 → matched/alias. (2) 회원 실명 정규화 정확일치: 단일 → matched/exact, 복수 → ambiguous(candidates=충돌 회원, score 1). (3) 부분포함(양방향) → unmatched + candidates(score 0.6, 짧은 쪽 길이 비율로 가산). (4) 없음 → unmatched, candidates=[].

```ts
// lib/dues/match-payer.ts
import { normalizePayerName } from "@/lib/dues/normalize-payer-name";

export type MemberRef = { memId: string; name: string };
export type AliasRef = { rawNameNorm: string; memId: string };
export type MatchCandidate = { memId: string; name: string; score: number };
export type MatchStatus = "matched" | "ambiguous" | "unmatched";
export type MatchResult = {
  status: MatchStatus;
  memId: string | null;
  via: "alias" | "exact" | null;
  candidates: MatchCandidate[];
};

const MAX_CANDIDATES = 5;

export function matchPayer(rawName: string, members: MemberRef[], aliases: AliasRef[]): MatchResult {
  const norm = normalizePayerName(rawName);
  if (!norm) return { status: "unmatched", memId: null, via: null, candidates: [] };

  // (1) 별칭 정확일치
  const aliasHits = [...new Set(aliases.filter((a) => a.rawNameNorm === norm).map((a) => a.memId))];
  if (aliasHits.length === 1) {
    const m = members.find((x) => x.memId === aliasHits[0]);
    return {
      status: "matched", memId: aliasHits[0], via: "alias",
      candidates: m ? [{ memId: m.memId, name: m.name, score: 1 }] : [],
    };
  }

  // 회원 정규화 캐시
  const normed = members.map((m) => ({ ...m, norm: normalizePayerName(m.name) }));

  // (2) 실명 정확일치
  const exact = normed.filter((m) => m.norm === norm);
  if (exact.length === 1) {
    return {
      status: "matched", memId: exact[0].memId, via: "exact",
      candidates: [{ memId: exact[0].memId, name: exact[0].name, score: 1 }],
    };
  }
  if (exact.length > 1) {
    return {
      status: "ambiguous", memId: null, via: null,
      candidates: exact.map((m) => ({ memId: m.memId, name: m.name, score: 1 })),
    };
  }

  // (3) 부분 포함(양방향) → 후보 제시(자동 확정 안 함)
  const partial = normed
    .filter((m) => m.norm.length > 0 && (norm.includes(m.norm) || m.norm.includes(norm)))
    .map((m) => {
      const shorter = Math.min(m.norm.length, norm.length);
      const longer = Math.max(m.norm.length, norm.length);
      return { memId: m.memId, name: m.name, score: 0.6 + 0.4 * (shorter / longer) };
    })
    .sort((a, b) => b.score - a.score)
    .slice(0, MAX_CANDIDATES);

  return { status: "unmatched", memId: null, via: null, candidates: partial };
}
```

```ts
import type { MatchStatus } from "@/lib/dues/match-payer";

export type Bucket = "autoDone" | "needsReview" | "excluded";

export function bucketOf(input: {
  io: "deposit" | "withdrawal";
  itemCd: string;            // 'due' | 'expense' | 'other' | ...
  matchStatus: MatchStatus;
}): Bucket {
  if (input.io === "withdrawal") return "excluded";
  if (input.itemCd === "other" || input.itemCd === "expense") return "excluded";
  if (input.itemCd === "due" && input.matchStatus === "matched") return "autoDone";
  return "needsReview";
}
```

  동작: 각 item의 `rawName`을 정규화 → `fee_payer_alias`에 upsert(`onConflict: team_id,raw_name_norm`), `mem_id`·`last_used_at` 갱신. 빈 정규화는 스킵.

```ts
// app/actions/dues/learn-aliases.ts
"use server";

import { dayjs } from "@/lib/dayjs";

import { withAdmin } from "@/lib/actions/auth";
import { getRequestTeamContext } from "@/lib/queries/request-team";
import { createAdminClient } from "@/lib/supabase/admin";

import { normalizePayerName } from "@/lib/dues/normalize-payer-name";

export type AliasLearnItem = { rawName: string; memId: string };

export async function learnAliases(items: AliasLearnItem[]) {
  return withAdmin(async ({ member }) => {
    const { teamId } = await getRequestTeamContext();
    const db = createAdminClient();

    const rows = items
      .map((it) => ({ norm: normalizePayerName(it.rawName), memId: it.memId }))
      .filter((r) => r.norm.length > 0)
      .map((r) => ({
        team_id: teamId,
        raw_name_norm: r.norm,
        mem_id: r.memId,
        last_used_at: dayjs().toISOString(),
        created_by: member.id,
        vers: 0,
        del_yn: false,
      }));

    if (!rows.length) return { ok: true as const, message: null, learned: 0 };

    const { error } = await db
      .from("fee_payer_alias")
      .upsert(rows, { onConflict: "team_id,raw_name_norm" });

    if (error) return { ok: false as const, message: "별칭 저장에 실패했습니다.", learned: 0 };
    return { ok: true as const, message: null, learned: rows.length };
  });
}
```

```ts
  const db = createAdminClient();

  const { data: rels } = await db.from("team_mem_rel").select("mem_id").eq("team_id", teamId).eq("vers", 0).eq("del_yn", false);
  const memIds = (rels ?? []).map((r) => r.mem_id);
  const { data: mems } = memIds.length
    ? await db.from("mem_mst").select("mem_id, mem_nm").in("mem_id", memIds).eq("vers", 0).eq("del_yn", false)
    : { data: [] as { mem_id: string; mem_nm: string }[] };
  const memberRefs: MemberRef[] = (mems ?? []).map((m) => ({ memId: m.mem_id, name: m.mem_nm }));

  const { data: aliasRows } = await db.from("fee_payer_alias").select("raw_name_norm, mem_id").eq("team_id", teamId).eq("del_yn", false);
  const aliasRefs: AliasRef[] = (aliasRows ?? []).map((a) => ({ rawNameNorm: a.raw_name_norm, memId: a.mem_id }));

  const { data: rows } = await db
    .from("fee_txn_hist")
    .select("txn_id, txn_dt, txn_tm, txn_amt, txn_io_enm, raw_name, mem_id, match_st_cd, fee_item_cd")
    .eq("team_id", teamId).eq("is_cfm_yn", false).eq("del_yn", false)
    .order("txn_dt", { ascending: true });

  const txns: InboxTxn[] = (rows ?? []).map((r) => {
    const match = matchPayer(r.raw_name, memberRefs, aliasRefs);
    const matchStatus = (r.match_st_cd ?? match.status) as InboxTxn["matchStatus"];
    const bucket = bucketOf({ io: r.txn_io_enm as "deposit" | "withdrawal", itemCd: r.fee_item_cd ?? "other", matchStatus });
    return {
      txnId: r.txn_id, txnDt: r.txn_dt, txnTm: r.txn_tm, amt: r.txn_amt,
      rawName: r.raw_name, memId: r.mem_id, matchStatus,
      feeItemCd: r.fee_item_cd, bucket, candidates: match.candidates,
    };
  });

  return { members: memberRefs.map((m) => ({ memId: m.memId, name: m.name })), txns };
}

export type LedgerRow = { memId: string; name: string; balance: number; status: "미납" | "정상" | "예치"; months: number };

export async function getDuesLedger(): Promise<{ rows: LedgerRow[]; summary: { unpaid: number; ok: number; prepaid: number } }> {
  const { teamId } = await getRequestTeamContext();
  const db = createAdminClient();

  const { data: policies } = await db
    .from("fee_policy_cfg").select("monthly_fee_amt").eq("team_id", teamId).eq("vers", 0).eq("del_yn", false)
    .order("aply_stt_dt", { ascending: false }).limit(1);
  const monthly = policies?.[0]?.monthly_fee_amt ?? 2000;

  const { data: snaps } = await db
    .from("fee_mem_bal_snap")
    .select("mem_id, bal_amt, mem_mst!fk_fee_mem_bal_snap__mem_mst(mem_nm)")
    .eq("team_id", teamId).eq("vers", 0).eq("del_yn", false);

  const rows: LedgerRow[] = (snaps ?? []).map((s) => {
    const bal = s.bal_amt;
    const memNm = Array.isArray(s.mem_mst) ? s.mem_mst[0]?.mem_nm : (s.mem_mst as { mem_nm: string } | null)?.mem_nm;
    const status: LedgerRow["status"] = bal < 0 ? "미납" : bal > 0 ? "예치" : "정상";
    return { memId: s.mem_id, name: memNm ?? "(이름없음)", balance: bal, status, months: Math.round(Math.abs(bal) / monthly) };
  }).sort((a, b) => a.balance - b.balance);

  return {
    rows,
    summary: {
      unpaid: rows.filter((r) => r.status === "미납").length,
      ok: rows.filter((r) => r.status === "정상").length,
      prepaid: rows.filter((r) => r.status === "예치").length,
    },
  };
}
```

> `mem_mst!fk_...` 임베드 별칭은 실제 FK 제약명을 확인해 맞춘다(`fee_mem_bal_snap`의 FK는 `fk_fee_mem_bal_snap__mem_mst`). 임베드가 어려우면 mem_id 목록으로 별도 조회 후 JS 조인.

`inbox-client.tsx`의 핵심 구조(완전한 동작 코드). 분류값 매핑: 회비→`due`, 프로젝트→`event_fee`, 제외→`other`.

```tsx
// app/(info)/admin/dues/inbox/inbox-client.tsx
"use client";

import { useMemo, useState, useTransition } from "react";

import { dayjs } from "@/lib/dayjs";
import { Button } from "@/components/ui/button";
import { H2, Body, Caption, Micro } from "@/components/common/typography";
import type { InboxTxn } from "@/lib/queries/dues";
import { confirmAndRecalc } from "@/app/actions/dues/confirm-and-recalc";

type Decision = {
  memId: string | null;
  itemCd: "due" | "event_fee" | "other"; // 회비 / 프로젝트 / 제외
  remember: boolean;
};

const ITEM_LABEL: Record<Decision["itemCd"], string> = { due: "회비", event_fee: "프로젝트", other: "제외" };

export function InboxClient({ members, txns }: { members: { memId: string; name: string }[]; txns: InboxTxn[] }) {
  const review = useMemo(() => txns.filter((t) => t.bucket === "needsReview"), [txns]);
  const autoDone = txns.filter((t) => t.bucket === "autoDone");
  const excluded = txns.filter((t) => t.bucket === "excluded");

  const [decisions, setDecisions] = useState<Record<string, Decision>>(() =>
    Object.fromEntries(review.map((t) => [t.txnId, {
      memId: t.candidates[0]?.memId ?? null,
      itemCd: "due" as const,
      remember: false,
    }])),
  );
  const [pending, startTransition] = useTransition();
  const [msg, setMsg] = useState<string | null>(null);

  const allDecided = review.every((t) => {
    const d = decisions[t.txnId];
    // 회비는 회원 매칭 필수, 프로젝트/제외는 회원 불필요
    return d && (d.itemCd !== "due" || d.memId);
  });

  function setDecision(txnId: string, patch: Partial<Decision>) {
    setDecisions((prev) => ({ ...prev, [txnId]: { ...prev[txnId], ...patch } }));
  }

  function onSubmit() {
    const items = review.map((t) => {
      const d = decisions[t.txnId];
      return { txnId: t.txnId, feeItemCd: d.itemCd, memId: d.itemCd === "due" ? d.memId : null };
    });
    const aliasLearn = review
      .map((t) => ({ t, d: decisions[t.txnId] }))
      .filter(({ d }) => d.remember && d.itemCd === "due" && d.memId)
      .map(({ t, d }) => ({ rawName: t.rawName, memId: d.memId! }));
    const recalcMemIds = [...new Set(items.filter((i) => i.memId).map((i) => i.memId!))];

    startTransition(async () => {
      const res = await confirmAndRecalc({ items, aliasLearn, recalcMemIds });
      setMsg(res.ok ? `확정 ${res.confirmed}건 · 재계산 ${res.recalculated}명 완료` : res.message);
    });
  }

  return (
    <div className="flex flex-col gap-0">
      {/* 헤더/요약 */}
      <div className="flex flex-col gap-4 px-6 pb-6">
        <H2>거래내역 처리</H2>
        <Caption>
          ✅ 자동완료 {autoDone.length} · ⚠️ 확인필요 {review.length} · 🚫 제외 {excluded.length}
        </Caption>

        {/* 확인필요 카드 큐 */}
        <div className="flex flex-col gap-3">
          {review.map((t) => {
            const d = decisions[t.txnId];
            return (
              <div key={t.txnId} className="rounded-2xl border border-border p-4 flex flex-col gap-3">
                <div className="flex items-center justify-between">
                  <Body className="font-semibold">{t.amt.toLocaleString()}원</Body>
                  <Micro>{dayjs(t.txnDt).format("YY.MM.DD")} · {t.rawName}</Micro>
                </div>

                {/* 후보 회원 */}
                {d.itemCd === "due" && (
                  <div className="flex flex-wrap gap-2">
                    {t.candidates.length === 0 && <Caption>후보 없음 — 직접 선택</Caption>}
                    {t.candidates.map((c) => (
                      <Button key={c.memId} size="sm"
                        variant={d.memId === c.memId ? "default" : "outline"}
                        onClick={() => setDecision(t.txnId, { memId: c.memId })}>
                        {c.name} {Math.round(c.score * 100)}%
                      </Button>
                    ))}
                    <select className="rounded-md border border-border px-2 text-sm"
                      value={d.memId ?? ""} onChange={(e) => setDecision(t.txnId, { memId: e.target.value || null })}>
                      <option value="">직접 선택…</option>
                      {members.map((m) => <option key={m.memId} value={m.memId}>{m.name}</option>)}
                    </select>
                  </div>
                )}

                {/* 분류 */}
                <div className="flex gap-2">
                  {(["due", "event_fee", "other"] as const).map((cd) => (
                    <Button key={cd} size="sm" variant={d.itemCd === cd ? "default" : "outline"}
                      onClick={() => setDecision(t.txnId, { itemCd: cd })}>
                      {ITEM_LABEL[cd]}
                    </Button>
                  ))}
                </div>

                {/* 기억하기 */}
                {d.itemCd === "due" && d.memId && (
                  <label className="flex items-center gap-2">
                    <input type="checkbox" checked={d.remember}
                      onChange={(e) => setDecision(t.txnId, { remember: e.target.checked })} />
                    <Caption>이 이름 기억하기 (다음부터 자동)</Caption>
                  </label>
                )}
              </div>
            );
          })}
        </div>

        {msg && <Caption className="text-foreground">{msg}</Caption>}
        <Button disabled={!allDecided || pending || review.length === 0} onClick={onSubmit}>
          {pending ? "처리 중…" : `${review.length}건 확정 + 회비 재계산`}
        </Button>
      </div>
    </div>
  );
}
```

> 묶음(같은 날·같은 금액 일괄)·키보드 단축키는 위 구조 위에 얹는 향상이다. 묶음: `review`를 `${txnDt}|${amt}`로 group해 2건 이상이면 그룹 헤더에 "일괄: [회비][프로젝트][제외]" 버튼을 두고, 클릭 시 그룹 내 모든 `txnId`에 `setDecision(..., { itemCd })`. 키보드: 카드에 `onKeyDown`으로 `1/2/3`→분류, `Enter`→다음 카드 focus. 두 향상은 Step 4에서 추가.
